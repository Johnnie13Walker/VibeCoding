# Блок `Просрочки по менеджерам`

## Назначение

Показать, у каких менеджеров подтверждённо сорвана дисциплина по задачам, и где уже нужен управленческий контроль.

## Что считается подтверждённой просрочкой

- задача не закрыта;
- у задачи есть `deadline`;
- `deadline < now` в МСК;
- задача привязана к активной сделке;
- можно надёжно определить менеджера сделки.

## Правила блока

- Показывать только менеджеров с реальной просрочкой.
- Не выводить менеджеров с `0`.
- По каждому менеджеру обязательно давать:
  - имя;
  - количество просрочек;
  - максимальную просрочку;
  - `1-3` коротких примера.
- Примеры лучше вести от сделки: ссылка на сделку + короткое название задачи.
- Если просрочек нет, выводить короткую заглушку.
- Если источник задач или привязка к CRM не подтверждены, писать ограничение, а не имитировать пустой результат.

## Эталонный формат

```text
🗂 Просрочки по менеджерам
• {manager_name} — {overdue_count} просрочки
макс просрочка: {max_days_overdue} дн.
примеры: <a href="{deal_url_1}">{deal_title_1}</a> → {task_title_1}; <a href="{deal_url_2}">{deal_title_2}</a> → {task_title_2}
```

## Если просрочек нет

```text
🗂 Просрочки по менеджерам: нет
```

## Если источник не подтверждён

```text
🗂 Просрочки по менеджерам
• Ограничение: нет подтверждённого доступа к задачам Bitrix24 или к привязке задач к сделкам.
```
