# Плейбук: как анализировать риски по воронке

## Какие сигналы считать рисками

- сделка под риском;
- сделка без движения;
- сделка без `следующего шага`;
- сделка без коммуникации;
- поздняя стадия без прогресса;
- крупная сумма, у которой уже есть хотя бы один сильный негативный сигнал.

## Как смотреть на сделки под риском

- сначала определить основной риск;
- потом понять, влияет ли он на деньги, срок или управляемость;
- не считать отдельным управленческим риском любую грязь в CRM;
- поднимать наверх только то, что требует вмешательства.

## Как смотреть на сделки без движения

- это сигнал потери темпа;
- особенно опасен на поздней стадии и на крупной сумме;
- ранняя маленькая сделка без движения не всегда должна попадать в управленческий фокус;
- если сигнал повторяется по одной сделке, приоритет повышается.

## Как смотреть на сделки без `следующего шага`

- это один из самых ранних сигналов потери контроля;
- без `следующего шага` сделка может ещё не выглядеть мёртвой, но уже не управляется;
- поздняя стадия без `следующего шага` требует быстрого вмешательства;
- крупная сумма без `следующего шага` должна попадать в денежный блок как риск.

## Как смотреть на сделки без коммуникации

- считать только подтверждённую внешнюю коммуникацию;
- рабочий наследованный порог для запущенной сделки — `> 14` дней без коммуникации;
- этот порог зафиксирован в источниках основного контура, но для текущего рабочего контура отдельно не подтверждён;
- поздняя или крупная сделка без коммуникации — сильный риск.

## Как не дублировать один и тот же риск в пяти местах отчёта

- для каждой сделки выбирать один основной риск;
- в `Состоянии воронки` показывать только агрегаты;
- в `Деньгах` показывать только денежный угол того же сигнала;
- в `Фокусе РОПа` показывать короткий приоритет, а не список всех риск-сделок;
- если у одной сделки несколько проблем, описывать корневой риск, а не все симптомы сразу.

## Как выбирать только действительно важные сигналы

- приоритетнее сделки с крупной суммой;
- приоритетнее поздние стадии;
- приоритетнее сделки с несколькими сигналами сразу;
- приоритетнее сигналы, требующие вмешательства РОПа или руководителя;
- не поднимать ранние слабые хвосты без управленческого смысла.

## Что делать, если данных не хватает

- не придумывать риск;
- не объявлять сделку проблемной без подтверждённого сигнала;
- если используется наследованная эвристика, помечать это;
- если нет подтверждённых коммуникаций, подтверждённых задач или подтверждённой карты стадий, писать ограничение прямо.
