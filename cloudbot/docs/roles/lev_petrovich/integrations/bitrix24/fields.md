# Ключевые поля Bitrix24

## Назначение файла

Это практический список полей, которые нужны для отчётов и сигналов роли Льва Петровича. Список собран по локально подтверждённым legacy-источникам, но live runtime `Bitrix24` в текущем workspace не подтверждён.

## Поля по сделке

| Поле | Зачем нужно | Где используется | Критично | Что делать, если поля нет |
| --- | --- | --- | --- | --- |
| `ID сделки` | однозначная идентификация сделки | ссылки, дедупликация, примеры | да | без поля сделку нельзя корректно связать с другими сущностями |
| `Название сделки` | читаемое название для отчётов | `Фокус РОПа`, примеры, ссылки | да | использовать краткий безопасный fallback, но помечать ограничение |
| `Сумма` | денежный объём сделки | `Воронка в работе`, детальные риск-списки, риск-суммы | да | денежные строки отчёта становятся неполными, это надо писать явно |
| `Стадия` | позиция сделки в воронке | `Воронка`, риск, high probability | да | использовать только подтверждённую нормализацию или помечать ограничение |
| `Ответственный` | владелец сделки | агрегация по менеджерам, примеры | да | без ответственного manager-level выводы недостоверны |
| `Источник` | канал входа сделки | поток, вспомогательная аналитика | нет | можно опустить из отчёта, но не подменять значением |
| `Дата создания` | возраст сделки | поток, длинный цикл, динамика | нет | использовать только если подтверждена |
| `Дата последнего изменения` | последний общий сдвиг карточки | proxy движения | средне | при отсутствии не строить жёсткий вывод о движении |
| `Следующий шаг` | явный план по сделке | `Воронка в работе`, `⚠️ Сделки без следующего шага`, `Фокус РОПа` | да | использовать только подтверждённый proxy и помечать ограничение |
| `Дата следующего действия` | срок следующего шага | `Воронка в работе`, `⚠️ Сделки без следующего шага`, приоритет | да | без поля нельзя жёстко утверждать, что шаг не зафиксирован |
| `Признак высокой вероятности` | вероятность закрытия | `Высокая вероятность` в обоих блоках | средне | использовать рабочую эвристику только с пометкой |
| `Признак риска` | явный риск-флаг или производный признак | `Под риском`, краткая сводка | средне | вычислять через подтверждённую логику, иначе не выдавать за факт |
| `Признак коммуникации / дата последней коммуникации` | понять, есть ли контакт с клиентом | `Запущенные сделки`, риск коммуникации | да | без поля нельзя подтверждать запущенные сделки |
| `Признак отсутствия движения / дата последнего движения` | понять, стоит ли сделка | `Воронка в работе`, риск по воронке | да | без поля или proxy нельзя подтверждать зависание |

## Поля по задачам

| Поле | Зачем нужно | Где используется | Критично | Что делать, если поля нет |
| --- | --- | --- | --- | --- |
| `ID задачи` | идентификация задачи | дедупликация, ссылки, примеры | да | без ID задача не может быть надёжно обработана |
| `Название задачи` | читаемое описание проблемы | `Просрочки`, `Фокус РОПа` | да | использовать сокращённый fallback только при явной необходимости |
| `Ответственный` | кто ведёт задачу | агрегация и приоритет | да | без ответственного блок по менеджерам неполный |
| `Привязка к сделке` | связать задачу с sales-контуром | `Просрочки`, `Фокус РОПа` | да | без привязки задача не должна попадать в эти блоки |
| `Срок` | понять факт просрочки | `Просрочки`, приоритет | да | без срока нельзя считать просрочку |
| `Дата последнего переноса срока` | понимать свежесть проблемы | `Фокус РОПа`, deep dive | нет | если поля нет, не использовать этот угол анализа |
| `Число переносов срока` | ключевой критерий `Фокуса РОПа` | `Фокус РОПа` | да | без поля нельзя подтверждать попадание задачи в блок |
| `Статус задачи` | исключить закрытые задачи | `Просрочки`, `Фокус РОПа` | да | без статуса список будет шумным |

## Поля по коммуникациям

| Поле | Зачем нужно | Где используется | Критично | Что делать, если поля нет |
| --- | --- | --- | --- | --- |
| `Дата последнего касания` | измерять тишину по сделке | `Воронка в работе`, `🚨 Сделки без коммуникации > 14 дн.`, риск-блоки | да | без этого нельзя подтверждать отсутствие коммуникации |
| `Тип коммуникации` | отделять звонок, сообщение, письмо | уточнение сигнала и качества контакта | нет | можно оставить только сам факт коммуникации |
| `Наличие/отсутствие коммуникации за период` | считать паузу `> 14` дней | `Воронка в работе`, `🚨 Сделки без коммуникации > 14 дн.`, риски | да | при отсутствии данных нужен явный дисклеймер |

## Рабочее правило

- если поле критично и не подтверждено, это ограничение отчёта;
- если поле может быть заменено proxy, proxy допустим только с явной пометкой;
- отсутствие поля нельзя маскировать выдуманным значением.
