# Bitrix24 для роли Льва Петровича

## Зачем нужна эта папка

Здесь собран рабочий контур `Bitrix24`, который нужен виртуальному коммерческому директору. Это не полная документация всей интеграции Cloudbot, а только sales-срез роли Льва Петровича.

## Что здесь хранится

- список сущностей, которые важны для роли;
- рабочий список полей для отчётов и сигналов;
- каркас карты стадий воронки;
- маппинги логики сигналов для `Sales Copilot`.

## Какую роль Bitrix24 играет в работе коммерческого директора

`Bitrix24` — главный потенциальный источник сделок, задач, коммуникаций, стадий, ответственных и сумм. Без него невозможно собрать production-версию `Sales Copilot`.

## Какие сущности и данные должны быть описаны

- сделки;
- лиды, если они реально участвуют в sales-контуре;
- контакты и компании;
- задачи;
- активности и коммуникации;
- ответственные менеджеры;
- стадии сделки;
- источники сделок;
- суммы сделок.

## Что уже может быть известно

- в каноническом runtime `../../assistant/engineer` есть документированные sales-источники, поля и рабочие эвристики;
- локально подтверждены legacy docs, tests и code paths, связанные с `Sales Copilot`;
- в текущем workspace есть только knowledge-слой и migration notes.

## Чего пока может не хватать

- live-подключения `Bitrix24`, подтверждённого в runtime;
- production-карты полей;
- production-карты стадий;
- подтверждённой логики задач, просрочек и коммуникаций на live-данных.

## Что здесь должно появиться позже

- точная карта сущностей;
- карта полей;
- карта стадий;
- правила расчёта сигналов;
- правила отбора сделок в отчёт;
- технические детали live-интеграции, если они будут подтверждены.

## Важное ограничение

Подключение `Bitrix24` в live runtime для этого workspace не подтверждено. Поэтому документы в этой папке фиксируют рабочий каркас и локально проверенные legacy-источники, а не production-факт подключения.
