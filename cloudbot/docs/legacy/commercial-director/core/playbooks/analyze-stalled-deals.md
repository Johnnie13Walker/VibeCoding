# Плейбук: как анализировать сделки без движения и зависшие сделки

## Назначение анализа

Нужно отличать обычную длинную сделку от сделки, по которой реально потерян темп.

## Как отличать «без движения» от просто длинной сделки

- длинная сделка сама по себе не проблема, если по ней есть подтверждённый прогресс;
- сделка `без движения` — это сделка, где нет подтверждённого сдвига по ключевым событиям за рабочий порог;
- рабочий наследованный порог, найденный в источниках основного контура, — `inactive_days >= 5`, но отдельного подтверждения этого порога пока нет;
- если подтверждённых данных по движению нет, нельзя называть сделку зависшей как факт.

## Какие признаки тревожны

- поздняя стадия без движения;
- крупная сумма без движения;
- отсутствие движения одновременно с отсутствием `следующего шага`;
- отсутствие движения одновременно с отсутствием коммуникации;
- повторяющийся простой по одной и той же сделке из отчёта в отчёт.

## Как показывать в отчёте

- в `Состоянии воронки` давать только агрегат `Без движения: количество`;
- в денежном блоке не повторять эту строку как отдельную операционную проблему;
- денежный смысл сигнала раскрывать только там, где сделка попадает в `Под риском` или `Запущенные без коммуникации > 14 дн.`;
- детальный список зависших сделок нужен только если он реально помогает принять решение.

## Как не перегружать отчёт деталями

- не перечислять все сделки без движения;
- не смешивать сделки без движения, без `следующего шага` и без коммуникации в один длинный список;
- оставлять только действительно важные сигналы;
- вывод по менеджерам делать только если есть повторяющийся паттерн.

## Как отделять операционный риск от денежного риска

- операционный риск: растёт число сделок без движения;
- денежный риск: без движения стоят крупные или поздние сделки;
- если сделка зависла и при этом попала в `Под риском`, в отчёте нужен один главный сигнал, а не пять дублей;
- если движение не подтверждено источником, это должно быть отмечено явно.
