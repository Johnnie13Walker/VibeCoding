# Плейбук: как анализировать сделки без следующего шага

## Зачем нужен анализ

Сделка без следующего шага быстро теряет управляемость. Это один из самых ранних сигналов проблемы по воронке.

## Какие данные нужны

- активные сделки в работе;
- название сделки;
- сумма;
- стадия;
- ответственный менеджер;
- явное поле `следующего шага`, если оно есть;
- дата следующего действия, если она есть;
- дата последнего движения;
- дата последней коммуникации.

## Как определять сделки без `следующего шага`

- в приоритете использовать явное поле `следующего шага` или дату следующего действия;
- если таких полей нет, допускается только подтверждённый замещающий признак из наследованной логики;
- закрытые, выигранные и проигранные сделки исключаются;
- если логика построена на замещающем признаке, это нужно помечать как ограничение.

## Когда это уже риск

- любая активная сделка без `следующего шага` уже операционный сигнал;
- поздняя стадия без `следующего шага` — усиленный риск;
- крупная сумма без `следующего шага` — денежный риск;
- отсутствие `следующего шага` вместе с отсутствием движения или коммуникации усиливает приоритет.

## Как показывать в отчёте

- в `Состоянии воронки` строкой `Без следующего шага: количество / сумма`;
- в `Деньгах` строкой `Без следующего шага: количество / сумма`;
- без повторного списка тех же сделок внутри обоих блоков.

## Как не дублировать соседние блоки

- в операционном блоке это сигнал потери контроля;
- в денежном блоке это деньги без понятного следующего действия;
- один и тот же сигнал можно показать в двух блоках, но смысл блока должен оставаться разным;
- не нужно дублировать одинаковую детализацию по сделкам.

## Какие управленческие выводы делать

- рост количества таких сделок означает слабую CRM-дисциплину;
- рост суммы по таким сделкам означает риск по выручке;
- концентрация сигнала у одного менеджера означает локальную проблему качества ведения;
- поздняя стадия без `следующего шага` требует немедленного вмешательства.
