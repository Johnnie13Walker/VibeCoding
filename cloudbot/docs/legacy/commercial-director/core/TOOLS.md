# Источники и ограничения

## Назначение файла

Файл фиксирует допустимые источники данных, рабочую зону роли и ограничения. Всё, что здесь не подтверждено, не должно подаваться как доступный live-источник.

## Приоритетные источники

- `Bitrix24`;
- CRM-данные по сделкам;
- менеджеры и ответственные;
- задачи;
- коммуникации;
- стадии воронки;
- Telegram-шаблоны и примеры отчётов;
- внутренние `playbooks`, `prompts` и `templates` этого контура;
- подтверждённые migration source из `../../assistant/engineer`, если нужен controlled extraction.

## Что может быть доступно уже сейчас

- локальные файлы этого контура `commercial-director/`;
- Telegram delivery через shared runtime `../../assistant/engineer`, если он уже подтверждён отдельно от этой папки;
- `web search`, если он включён глобально;
- runtime tools `OpenClaw`, если они доступны в текущем окружении;
- материалы из `../../assistant/engineer` как источник для extraction и сверки.

## Что нужно отдельно проверить

- подключён ли `Bitrix24` в live runtime;
- доступны ли реальные поля CRM;
- доступны ли sales-скрипты и их актуальные точки входа;
- доступны ли данные по задачам;
- доступны ли данные по коммуникациям;
- доступны ли данные по просрочкам;
- подтверждён ли отдельный Telegram routing роли;
- подтверждён ли отдельный runtime именно для `commercial-director`.

## Что разрешено

- использовать локальные документы, шаблоны, prompts, playbooks и примеры этого контура;
- использовать подтверждённые материалы из `../../assistant/engineer` как источник для controlled extraction;
- формализовать структуру роли, отчётов и сигналов;
- строить рабочие каркасы по `Bitrix24`, если live runtime не подтверждён, но reference-источник в `../../assistant/engineer` найден;
- явно фиксировать ограничения и пробелы в данных.

## Что запрещено

- personal memory founder;
- personal assistant контекст;
- DevOps и серверные runbooks;
- shell и infra как базовая зона роли;
- CFO-контур;
- выдача неподтверждённых live-источников за рабочие;
- хранение секретов, webhook, токенов и runtime-конфигов в git.

## Отдельный принцип

Если источник данных не подтверждён, нельзя делать вид, что он есть. В этом случае допустимы только:

- явная пометка ограничения;
- рабочий каркас документа;
- ссылка на legacy-источник, если он подтверждён локально;
- список того, что нужно подтвердить отдельно.
