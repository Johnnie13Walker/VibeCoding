# Блок `Фокус РОПа`

## Назначение

Короткий жёсткий управленческий блок для коммерческого директора. Его задача: не пересказывать весь отчёт, а быстро ответить на три вопроса:

- где потерян контроль;
- где сегодня можно взять деньги;
- что именно РОП должен сделать сегодня руками.

Блок отправляется отдельным Telegram-сообщением сразу после `Sales Copilot`.

## Что уже есть в Sales Copilot и не должно дублироваться

- `📈 Воронка в работе` уже держит полный срез по количеству, сумме, деньгам под риском и составу воронки.
- `⚠️ Сделки без следующего шага` уже держат полный список сделок, где потерян понятный следующий ход.
- `🚨 Сделки без коммуникации > 14 дн.` уже держат полный список сделок с длинной тишиной.
- `🗂 Просрочки по менеджерам` уже держат полный список просрочек, их количество, глубину и примеры.
- `📞 Коммуникации за вчера`, если блок есть рядом, уже держат дневную активность менеджеров.
- `Фокус РОПа` забирает только отклонение, приоритет и действие на сегодня.

## Что убрать из старого формата

- голые агрегаты из воронки вроде `под риском`, `без следующего шага`, `без коммуникации`, если рядом уже есть тот же срез;
- полный список просрочек по людям;
- полные списки сделок без следующего шага и без коммуникации;
- отдельный подзаголовок со списком `сделок без следующего шага`: этот блок уже есть в `Sales Copilot`;
- таблицу вчерашних коммуникаций;
- повтор одной и той же сделки без нового управленческого смысла.

## Что оставлять только сокращённо

- концентрацию проблем по менеджеру: одной строкой `у кого потерян контроль` вместо длинного перечня сделок;
- деньги под риском: не общим итогом, а только через `2-3` живые сделки, где есть действие на сегодня;
- тишину и просрочки: только как основание для управленческого сигнала, а не как второй полный список;
- ложный прогресс: только там, где статус сделки нужно пересобрать уже сегодня.

## Что нужно добавить, чтобы блок был управленческим

- где деньги ещё можно быстро вернуть в движение;
- где РОПу уже надо самому выйти на клиента;
- где продажа не доведена до обязательного следующего шага;
- кого именно РОП должен дожать сегодня;
- три конкретных действия РОПа на день;
- короткий позитивный итог по сотрудникам с `10/10` за вчера.

## Принцип сборки

1. Сначала сравнить кандидатов с соседними блоками `Sales Copilot`.
2. Полные дубли убрать, а повторяющийся смысл сжать до одной управленческой строки.
3. Приоритизировать сигналы по потере контроля, близости денег и необходимости личного входа РОПа.
4. Одной сделке давать один главный угол. Повтор сделки допустим только при новом управленческом смысле; третий повтор оставлять лишь для действительно критичной сделки.

## Что должно попасть в блок

- `Где потерян контроль`:
  - максимум `3-4` самых важных сигнала;
  - показывать только потерю управляемости по менеджеру или по группе сделок;
  - если у менеджера несколько похожих проблем, сжимать их в одну строку;
  - основания: нет `следующего шага`, поздняя стадия без движения, нет коммуникации, повторные переносы сроков.
- `💸 Где взять деньги сегодня`:
  - только `2-3` сделки, где деньги ещё живы и есть шанс быстро вернуть движение;
  - на строке должны быть сделка, сумма и одно понятное действие на сегодня;
  - сюда не попадают сделки, которые уже фактически брошены и требуют не дожима, а пересборки статуса.
- `☎️ Где РОПу самому связаться с клиентом`:
  - только `1-2` сделки;
  - сюда попадают только случаи, где обычного хода менеджера уже недостаточно;
  - причины должны быть явными: долгая тишина, поздняя стадия, большая сумма, менеджер уже не вывез, высокий риск потери.
- `🕳 Упущенные возможности`:
  - только сделки, где продажу не довели до обязательного шага;
  - ключевой провал: нет нормального перехода от брифа к защите КП;
  - базовые сценарии: после брифа прошло `7` дней без защиты КП, стадия `Бриф` без встреч, зависание между брифом и защитой КП.
- `Кого дожать сегодня`:
  - только `2-3` сотрудника;
  - ФИО всегда выделять жирным;
  - указывать, что именно нужно сделать сегодня и, если возможно, до какого времени.
- `Что РОП обязан сделать сегодня`:
  - ровно `3` коротких действия;
  - без воды и без повтора уже выведенных списков;
  - это должен быть синтез дня, а не копия строк из подпунктов.
- `🔥 Молодцы за вчера`:
  - блок не считает баллы сам;
  - он берёт готовую оценку `операционная вовлечённость за вчера`;
  - показывать только сотрудников с ровно `10/10`;
  - ФИО всегда выделять жирным;
  - если сотрудников с `10/10` нет, блок можно скрыть или дать нейтральную заглушку.

## Эталонный формат

```text
🎯 Фокус РОПа

Где потерян контроль:
• У <b>{manager_name_1}</b> {control_loss_signal_1}
• У <b>{manager_name_2}</b> {control_loss_signal_2}
• У <b>{manager_name_3}</b> {control_loss_signal_3}

💸 Где взять деньги сегодня:
• <a href="{deal_url_1}">{deal_title_1}</a> — <b>{deal_amount_1}</b> — {money_action_1}
• <a href="{deal_url_2}">{deal_title_2}</a> — <b>{deal_amount_2}</b> — {money_action_2}
• <a href="{deal_url_3}">{deal_title_3}</a> — <b>{deal_amount_3}</b> — {money_action_3}

☎️ Где РОПу самому связаться с клиентом:
• <a href="{escalation_deal_url_1}">{escalation_deal_1}</a> — <b>{escalation_amount_1}</b> — {escalation_reason_1}
• <a href="{escalation_deal_url_2}">{escalation_deal_2}</a> — <b>{escalation_amount_2}</b> — {escalation_reason_2}

🕳 Упущенные возможности:
• <a href="{opportunity_deal_url_1}">{opportunity_deal_1}</a> — {opportunity_gap_1} — {opportunity_reason_1}
• <a href="{opportunity_deal_url_2}">{opportunity_deal_2}</a> — {opportunity_gap_2} — {opportunity_reason_2}

Кого дожать сегодня:
• <b>{manager_name_3}</b> — {manager_action_1}
• <b>{manager_name_4}</b> — {manager_action_2}
• <b>{manager_name_5}</b> — {manager_action_3}

Что РОП обязан сделать сегодня:
• {action_item_1}
• {action_item_2}
• {action_item_3}

🔥 Молодцы за вчера
• <b>{top_employee_1}</b> — 10/10 👏🔥
• <b>{top_employee_2}</b> — 10/10 🚀💪
```

## Если часть данных пустая

- не выводить пустой подзаголовок;
- если полезных сигналов нет, дать короткую заглушку: `• Критичных управленческих отклонений на сегодня нет`;
- если блок `🔥 Молодцы за вчера` пуст, лучше скрыть его;
- если без него конец блока выглядит оборванным, можно написать: `• Лидеров с 10/10 по операционной вовлечённости за вчера нет`.

## Если источник не подтверждён

```text
🎯 Фокус РОПа
• Ограничение: нет подтверждённых данных по задачам, встречам, движению сделок или оценке операционной вовлечённости для управленческого блока.
```
