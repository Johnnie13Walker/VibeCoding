# Бэклог роли

## Назначение файла

Файл ведёт backlog роли `larisa_assistant`: что уже оформлено, что делается сейчас, что нужно следующим шагом и какие вопросы нельзя закрыть без ручного решения.

## Что уже сделано

- создан отдельный контур `assistant`;
- оформлены role-файлы Ларисы Ивановны;
- описаны границы personal assistant контура;
- подготовлен контракт агента `larisa_assistant`;
- подготовлен контракт workflow `personal_assistant`;
- подготовлены точки подключения к `calendar`, `todo`, `weather`, `news`, `search`;
- создан шаблон brief дня.

## Текущие задачи

- держать отдельную роль Ларисы Ивановны в изоляции от Льва Петровича;
- фиксировать структуру MVP brief дня;
- доформализовать команды и сценарии пользователя;
- подготовить безопасный переход от каркаса к runtime.

## Следующие шаги

- подтвердить live `calendar`;
- подтвердить live `todo`;
- определить источник weather по Москве;
- зафиксировать темы новостного brief;
- подключить безопасный сценарий создания встречи;
- решить, будет ли поиск идти через shared `web search` или отдельный skill;
- подготовить отдельный Telegram routing и расписание brief дня.

## Блокеры

- live-интеграции personal assistant контура пока не подтверждены;
- отдельная git-ветка для этого контура не может быть создана в текущей папке, пока она не находится внутри git-репозитория;
- пользовательские темы для новостей пока не перечислены;
- боевой runtime-routing на этом шаге не меняется.

## Вопросы на ручное решение

- какой календарь считается основным;
- какой task-provider считается основным;
- в какое время daily brief должен отправляться по МСК;
- какие темы входят в новостной блок по умолчанию;
- какой канал доставки используется для этого контура;
- должен ли personal assistant жить в отдельном runtime или на shared orchestrator.

## Принцип ведения файла

- фиксировать только задачи, относящиеся к Ларисе Ивановне;
- не тащить сюда CRM, продажи, финансы и DevOps;
- всё, что требует live-доступов или архитектурного решения, помечать явно;
- не считать каркас завершённым, пока не подтверждены интеграции и routing.
