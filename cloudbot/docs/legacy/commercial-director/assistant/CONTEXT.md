# Контекст роли

## Назначение файла

Файл собирает текущий контекст роли Ларисы Ивановны. На него должны опираться workflow, prompt, шаблоны brief дня и дальнейшая интеграция personal assistant контура в Cloudbot.

## Назначение daily brief

Daily brief Ларисы Ивановны — это короткий личный обзор дня пользователя. Его задача: быстро показать расписание, задачи, окна для фокуса, внешние условия дня и помочь собрать рабочий план без захода в продажи и CRM.

## Ключевые договорённости по структуре

- brief строится только по московскому времени;
- встречи и задачи идут раньше погоды и новостей;
- свободные окна считаются только между подтверждёнными встречами;
- внешний контекст не должен перекрывать личное расписание;
- рекомендации на день короткие и прикладные;
- ограничения источников указываются прямо в нужном блоке.

## Блок «Встречи»

Это основной блок расписания. В нём используются только подтверждённые встречи на текущий день.

Смысл блока:

- показать, во сколько начинается день;
- показать обязательные встречи;
- не скрывать перегруженные интервалы;
- не подменять пустой календарь догадками.

## Блок «Задачи»

Это блок личных задач на сегодня и просроченных задач.

Смысл блока:

- показать, что нужно закрыть сегодня;
- отдельно подсветить просроченное;
- не смешивать задачи с коммерческими поручениями из CRM;
- держать приоритеты короткими и понятными.

## Блок «Свободные окна»

Это операционный блок планирования дня.

Смысл блока:

- показать доступные интервалы между встречами;
- помочь пользователю понять, когда есть время на фокусную работу;
- не создавать окна, если календарь не подтверждён;
- не выдавать приблизительные интервалы за факт.

## Блок «Погода по Москве»

Это короткий внешний блок контекста.

Смысл блока:

- показать температуру и основные условия дня;
- предупредить о заметных изменениях погоды;
- не превращать блок в длинный метеоотчёт.

## Блок «Новости по темам»

Это короткий информационный блок.

Смысл блока:

- показать только выбранные пользователем темы;
- оставить `3-5` ключевых сигналов;
- не подменять новостной блок общим web search;
- при отсутствии подтверждённого источника явно писать ограничение.

## Блок «Фокус дня»

Это короткий управленческий итог personal assistant контура.

Смысл блока:

- собрать `3` практичных действия на день;
- не копировать целиком встречи и задачи;
- не уходить в стратегию продаж, CRM и финансы.

## Чего пока может не хватать

- live-подключения `calendar` и `todo` не подтверждены в этом контуре;
- отдельный news-профиль тем для пользователя ещё не зафиксирован;
- отдельный Telegram routing роли на этом шаге не настраивается;
- runtime-реализация workflow и provider adapters не переносится автоматически из legacy.
