## Принципы структуры
- Общие head-term-кластеры остаются на главной и hub-страницах услуг.
- Точные transactional-запросы выносятся в отдельные URL, чтобы не было каннибализации.
- Нишевые кластеры по стоматологии, косметологии и пластической хирургии разделяются на `SEO` и `рекламу`.
- Информационные запросы вида `контент план ...` не должны вести на коммерческие лендинги. Для них лучше отдельные статьи с CTA.

## Что не тянуть в коммерческие лендинги
- `контент план для стоматологии`
- `контент план для медицинской клиники`
- `контент план для стоматологической клиники`
- `контент для стоматологической клиники`

Для этих запросов лучше отдельные статьи в блоге с внутренними ссылками на коммерческие услуги.

## Практическая логика перелинковки
- С главной ссылки на: `marketing`, `seo`, `reklama-v-yandeks-direkt`, `razrabotka-saytov`, `upravlenie-reputaciej`.
- С `marketing` ссылки на: `marketingovyy-audit-kliniki`, `seo`, `reklama-v-yandeks-direkt`, `upravlenie-reputaciej`.
- С `seo` ссылки на все нишевые SEO-страницы.
- С `reklama-v-yandeks-direkt` ссылки на `reklama-stomatologii`, `reklama-kosmetologii`, `reklama-plasticheskoy-khirurgii`.
- С ORM-страниц ссылки между `upravlenie-reputaciej`, `reputatsiya-vracha`, `prodvizhenie-vrachey-na-prodoctorov`.
- Из контентных статей ссылки на ближайшие коммерческие страницы по смыслу.

## Риск каннибализации
- Не продвигать `агентство медицинского маркетинга` одновременно на главную и `/services/marketing/` как один и тот же primary query. Для главной оставить бренд + agency, для `/services/marketing/` оставить service intent.
- Не смешивать `реклама стоматологии` и `продвижение стоматологии` на одной странице. Первая — performance, вторая — SEO.
- Не смешивать `сайт для стоматологии` и `продвижение стоматологии`. Это разные этапы спроса и разные офферы.
- `управление репутацией клиники` и `репутация врача` должны быть разнесены по разным URL.
