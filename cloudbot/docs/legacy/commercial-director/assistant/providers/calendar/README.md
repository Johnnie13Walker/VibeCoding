# Provider `calendar`

## Роль

Точка подключения к календарю пользователя для personal assistant контура.

## Что нужно от provider

- получить встречи на дату;
- создать новую встречу;
- вернуть время начала и окончания;
- вернуть название, описание, участников и ссылку, если это поддерживается.

## Что использует provider

- workflow `personal_assistant`;
- сценарии `day_briefing` и `add_calendar_event`.

## Ограничения

- время всегда нормализуется в `Europe/Moscow`;
- неподтверждённая интеграция не считается live;
- CRM-календарь не должен становиться основным источником personal assistant контура без отдельного решения.
