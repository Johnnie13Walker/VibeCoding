# Provider `todo`

## Роль

Точка подключения к личным задачам пользователя.

## Что нужно от provider

- получить задачи на сегодня;
- получить просроченные задачи;
- вернуть приоритет, срок и краткое описание;
- поддержать будущую нормализацию задач для brief дня.

## Что использует provider

- workflow `personal_assistant`;
- сценарий `day_briefing`.

## Ограничения

- не смешивать личные задачи с CRM-задачами;
- неподтверждённая интеграция не считается live;
- при отсутствии сроков задача не должна искажать расчёт дневного фокуса.
