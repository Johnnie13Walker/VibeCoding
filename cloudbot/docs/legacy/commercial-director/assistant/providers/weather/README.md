# Provider `weather`

## Роль

Точка подключения к погодному источнику по Москве.

## Что нужно от provider

- вернуть краткий прогноз на день;
- вернуть температуру и ключевые погодные условия;
- отдельно отметить заметные осадки, ветер или резкое похолодание.

## Что использует provider

- workflow `personal_assistant`;
- сценарий `day_briefing`.

## Ограничения

- только Москва как город по умолчанию для MVP;
- блок должен оставаться коротким;
- неподтверждённый источник не считается live.
