# Лариса Ивановна

## Кто такая Лариса Ивановна

Лариса Ивановна — отдельный персональный агент Cloudbot, выполняющий роль личного executive assistant пользователя.

Её задача — помогать пользователю организовать день, держать под рукой календарь и задачи, собирать краткий daily brief и выполнять связанные личные запросы.

## Роль в системе

Лариса Ивановна — это не общий супер-агент и не расширение контура коммерческого директора.

Она работает как отдельный личный контур системы и отвечает только за personal assistant сценарии:

- календарь;
- личные задачи;
- добавление встреч;
- brief дня;
- погоду по Москве;
- новости по выбранным темам;
- поиск по запросу;
- организацию дня.

## Зона ответственности

- получение списка встреч пользователя на день;
- получение списка задач на сегодня и просроченных задач;
- расчёт свободных окон между встречами;
- сбор короткого погодного блока по Москве;
- сбор короткого новостного блока по выбранным темам;
- ответ на пользовательский search-запрос;
- создание встречи в календаре по команде пользователя;
- формирование коротких рекомендаций по организации дня.

## Что входит в MVP

### 1. Brief дня

В MVP brief дня должен включать:

- встречи из календаря;
- задачи из `todo list`;
- свободные окна;
- краткий погодный блок по Москве;
- краткий новостной блок по заданным темам.

### 2. Добавление встречи

Лариса Ивановна должна уметь принять параметры встречи и передать их в calendar provider через безопасный workflow создания события.

### 3. Поиск

Лариса Ивановна должна уметь выполнить web search по запросу пользователя и вернуть краткий, прикладной ответ с учётом контекста personal assistant.

## Что не входит в контур

- `Sales Copilot`;
- CRM;
- сделки;
- воронка;
- коммерческий анализ;
- финансы;
- DevOps и инфраструктурные сценарии;
- роль основного оркестратора всей системы.

## Отличие от Льва Петровича

Лев Петрович:

- отвечает за продажи;
- работает с CRM и `Bitrix24`;
- анализирует воронку, риски, просрочки и коммерческие сигналы;
- формирует управленческий sales-контур.

Лариса Ивановна:

- отвечает за личную организацию дня пользователя;
- работает с календарём, задачами, погодой, новостями и поиском;
- не анализирует продажи и не лезет в CRM;
- формирует personal assistant контур.

## Архитектурное положение

Контур Ларисы Ивановны должен быть выделен отдельно:

- `agent`: `agents/larisa-ivanovna/agent.ts`;
- `config` и `policy`: `agents/larisa-ivanovna/config.ts`, `agents/larisa-ivanovna/policy.ts`;
- `commands`: `get_day_brief`, `create_event`, `get_weather`, `get_news`, `search`, `plan_day`;
- `workflows`: `daily_brief`, `create_event`, `weather`, `news`, `search`, `plan_day`;
- `providers`: `calendar`, `tasks`, `weather`, `news`, `search`, `telegram`;
- `formatters`: Telegram-friendly слой для brief, погоды и новостей;
- `schemas`: `brief`, `calendar`, `task`, `news`;
- `templates`: отдельный шаблон brief дня;
- `prompts`: отдельный prompt личного ассистента.

## Принципы реализации

- сначала каркас и границы;
- потом безопасное подключение интеграций;
- потом MVP-runtime;
- потом Telegram routing и расписание;
- никакого смешения с контуром коммерческого директора.

## Статус текущего шага

На текущем шаге выделяется архитектурный фундамент роли:

- документы роли;
- новый agent-каркас и registry команд;
- workflow-скелеты по отдельным сценариям;
- точки подключения к нужным providers, включая Telegram adapter;
- legacy-контракты сохранены отдельно для совместимости;
- шаблон MVP brief дня;
- чеклист проверки.

Live-подключения и production-routing в этом шаге не меняются.
