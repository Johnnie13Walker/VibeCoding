# Cloudbot: стартовая схема для автономной работы Codex

Этот каталог подготовлен как безопасный стартовый каркас для проекта Cloudbot.

Здесь заложены:

- репозиторные правила для агента в `AGENTS.md`;
- план работ для фоновых задач в `docs/PLAN.md`;
- статусный журнал в `docs/STATUS.md`;
- чеклисты проверок в `docs/checklists/`;
- готовые промпты для Codex в `docs/prompts/`;
- описание GitHub workflow в `docs/workflows/`.

## Как использовать

1. Инициализировать git-репозиторий в этой папке или перенести файлы в существующий репозиторий Cloudbot.
2. Создать ветки `main` и `dev`.
3. Дополнить `docs/PLAN.md` реальными задачами проекта.
4. Дополнить `docs/checklists/health-check.md` конкретными сервисами, cron-задачами и интеграциями.
5. Запускать Codex в фоне по готовым промптам из `docs/prompts/`.

## Минимальный безопасный режим

Для начала лучше использовать два сценария:

1. Ночной проход по backlog без deploy.
2. Утренний health-check и статусный отчёт в `09:30 МСК`.

## Что нужно дописать в первую очередь

- реальные команды запуска и тестов;
- список systemd/docker сервисов;
- источники логов;
- обязательные ежедневные отчёты;
- список критичных ручных сценариев.
