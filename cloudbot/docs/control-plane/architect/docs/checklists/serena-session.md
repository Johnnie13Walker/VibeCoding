# Чеклист новой сессии с Serena

Использовать в новых сессиях Codex для задач по кодовой базе.

## Когда считать, что Serena реально используется

1. Сессия открыта в нужном контуре:
   - `/Users/pro2kuror/Desktop/Cloudbot/architect`
   - или `/Users/pro2kuror/Desktop/Cloudbot/engineer`
2. В начале сессии вызваны:
   - `serena.activate_project`
   - `serena.check_onboarding_performed`
   - `serena.initial_instructions`
3. Для изучения кода используются Serena-инструменты:
   - `get_symbols_overview`
   - `find_symbol`
   - `find_referencing_symbols`
4. Исходники не читаются целиком без необходимости через `cat` или `sed`.
5. Изменения в коде делаются точечно по символам или после архитектурного обзора, а не через слепую массовую замену.

## Быстрая фраза для новой задачи

`Сначала активируй Serena, проверь onboarding и работай через symbol-based tools.`

## Почему это экономит токены

- В контекст попадают только нужные символы, а не целые файлы.
- Быстрее находятся связи между функциями, классами и вызовами.
- Повторное объяснение структуры проекта сокращается за счет onboarding и memories.
- На больших репозиториях уменьшается объем лишнего кода в анализе и ответах.

## Когда Serena не обязательна как основной инструмент

- Для логов, markdown, yaml и других не-кодовых файлов.
- Для одноразовых shell-команд и простых проверок окружения.
- Для очень маленьких файлов, где чтение целиком дешевле и понятнее.
