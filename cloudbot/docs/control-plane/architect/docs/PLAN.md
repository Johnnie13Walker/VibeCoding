# PLAN

Этот файл — единственный источник задач для автономного фонового запуска Codex, если пользователь явно не указал другой backlog.

## Правила выполнения

1. Выполняй задачи сверху вниз.
2. Не начинай следующую задачу, пока не завершена текущая.
3. После каждого изменения обновляй `docs/STATUS.md`.
4. После каждого изменения запускай тесты и обязательные проверки.
5. Не делай deploy без прямого указания пользователя.
6. Если задача упирается в секреты, внешние доступы или риск для данных, остановись и зафиксируй блокер в `docs/STATUS.md`.
