# Коммерческий аудит рекламной кампании Belberry

Дата: 21 апреля 2026 г., МСК

Контекст: кампания `Товарная / МК / СПб, Мск` получила 2 853 клика, но по бизнес-контуру подтвержден только 1 целевой визит. Внутренние 495 конверсий Директа не считаются лидами до расшифровки целей.

## 1. Управленческий вывод

Проблема не в том, что Директ плохо настроен. Проблема в том, что бизнес не видит управляемую экономику воронки.

Сейчас есть трафик, есть дешевые внутренние события, но почти нет подтвержденных лидов. Это означает одно из пяти:

1. Покупается не тот трафик.
2. Оффер не конвертирует спрос в обращение.
3. Посадочная страница не доводит пользователя до заявки.
4. Аналитика не фиксирует реальные лиды.
5. Лиды появляются, но не попадают в управленческий контур CRM/продаж.

Пока не доказано, что 2 853 клика создали реальные заявки, бюджет нельзя масштабировать. Кампания должна оцениваться не по кликам, CTR и автоцелям, а по цепочке:

`расход -> клик -> лид -> квалифицированный лид -> встреча/бриф -> сделка -> выручка -> маржа`.

## 2. Почему опасно объединять поиск и сети в одной кампании

Поиск и сети - это разные типы спроса.

Поиск:

- пользователь уже формулирует потребность;
- можно понять интент через запрос;
- проще оценить соответствие оффера;
- обычно меньше объем, но выше управляемость;
- ключевой риск - дорогой клик и нерелевантные запросы.

Сети:

- пользователь не ищет услугу прямо сейчас;
- спрос чаще холодный или полутеплый;
- качество зависит от площадок, креативов, аудиторий и ретаргетинга;
- объем выше, но контроль ниже;
- ключевой риск - дешевые клики без коммерческого намерения.

Если поиск и сети объединены, коммерческий директор не может ответить на базовые вопросы:

- где реально потрачен бюджет;
- где возникли лиды;
- какой CPL отдельно на поиске и в сетях;
- какая доля кликов пришла от холодной аудитории;
- какой канал дает квалифицированные заявки;
- какой канал нужно отключить, если план продаж не выполняется.

Главный риск: бюджет начинает управляться алгоритмом, а не бизнес-логикой. Кампания может показывать красивые внутренние конверсии, но коммерческий директор не видит, какой сегмент приносит деньги.

## 3. Риски для управления бюджетом

### 3.1. Невозможно понять, что масштабировать

Если в одной кампании смешаны поиск, РСЯ, ретаргетинг, товарные фильтры и автотаргетинг, то хороший агрегированный показатель может скрывать убыточный сегмент.

Пример: ретаргетинг может давать дешевые события, а автотаргетинг съедать основной бюджет. В сумме кампания выглядит приемлемо, но бизнес теряет деньги.

### 3.2. Невозможно быстро остановить утечку

Когда CPL растет, нужно понимать, где резать бюджет. В смешанной кампании нельзя безопасно остановить только проблемный канал без риска задеть рабочий.

### 3.3. Алгоритм оптимизируется не на прибыль

Если в стратегии участвуют микроцели, алгоритм может покупать события, которые не имеют отношения к продажам: визиты, клики, просмотры, открытие формы. Для интерфейса это конверсии, для бизнеса - шум.

### 3.4. Отдел продаж не может дать обратную связь по сегментам

Если в CRM не передаются источник, кампания, группа, объявление, условие показа и цель, продажам нечего возвращать маркетингу. Нельзя сказать: "эта связка дает мусорные лиды", "эти заявки качественные", "этот канал приносит встречи".

## 4. Правильная структура кабинета для управляемого performance-маркетинга

Структура должна строиться не вокруг удобства интерфейса, а вокруг управляемости прибыли.

Минимальная правильная архитектура:

1. `Brand / Search / MSK-SPB`
2. `Generic / Search / Hot Demand / MSK`
3. `Generic / Search / Hot Demand / SPB`
4. `Medical Marketing / Search / MSK-SPB`
5. `Dental Marketing / Search / MSK-SPB`
6. `SMM / Search / MSK-SPB`
7. `SEO / Search / MSK-SPB`
8. `Competitors / Search / MSK`
9. `Retargeting / Network / Warm 7-30d`
10. `Retargeting / Network / Warm 31-90d`
11. `Lookalike / Network / Leads / MSK-SPB`
12. `Cold Network / Test / Separate Budget`
13. `Product / Network / MSK`
14. `Product / Network / SPB`

Каждая кампания должна иметь отдельный бюджет, отдельную цель оптимизации и отдельный управленческий KPI.

## 5. Сегменты, которые обязательно нужно разделить

Обязательно разделить:

- поиск и сети;
- Москва и Санкт-Петербург;
- бренд и общий спрос;
- горячий спрос и информационный спрос;
- медицинский маркетинг, стоматологии, SMM, SEO, performance;
- ретаргетинг и холодные аудитории;
- автотаргетинг и управляемые условия показа;
- товарные фильтры и ретаргетинг;
- мобильные и десктоп;
- лидовые цели и микроцели;
- новые пользователи и вернувшиеся;
- текущие клиенты/посетители и новая аудитория.

Разделение нужно не ради порядка. Оно нужно, чтобы можно было ответить:

- где CPL выше нормы;
- где лиды некачественные;
- где стоит снизить бюджет;
- где стоит увеличить бюджет;
- где проблема в оффере;
- где проблема в продажах.

## 6. Какие данные обязательно должны попадать в CRM

В CRM по каждому лиду должны попадать:

- дата и время лида в МСК;
- источник: Яндекс.Директ;
- кампания;
- ID кампании;
- группа;
- ID группы;
- объявление;
- ID объявления;
- условие показа;
- тип условия показа;
- поисковый запрос, если есть;
- площадка РСЯ, если есть;
- устройство;
- регион;
- UTM-метки;
- посадочная страница;
- форма/номер/мессенджер, через который пришел лид;
- цель Метрики;
- goalId;
- тип цели: форма, звонок, WhatsApp, Telegram, запись;
- стоимость клика;
- расход до лида;
- статус лида;
- причина отказа;
- квалификация лида;
- ответственный менеджер;
- дата первого контакта;
- время до первого контакта;
- результат контакта;
- создана ли сделка;
- сумма сделки;
- маржа;
- финальный статус: продажа, отказ, дубль, нецелевой, недозвон.

Без этих полей маркетинг не может управлять прибылью. Он управляет только трафиком.

## 7. Как должен выглядеть управленческий отчет

Отчет коммерческого директора должен начинаться не с CTR, а с денег.

Обязательные блоки:

1. Расход.
2. Реальные лиды.
3. CPL по реальным лидам.
4. Квалифицированные лиды.
5. CPQL.
6. Встречи/брифы.
7. Стоимость встречи.
8. Сделки.
9. CAC.
10. Выручка.
11. Маржа.
12. ROMI.
13. Окупаемость по сегментам.

Разрезы:

- поиск / сети;
- кампания;
- группа;
- объявление;
- условие показа;
- регион;
- устройство;
- посадочная;
- менеджер продаж;
- причина отказа;
- тип услуги.

Пример управленческой таблицы:

| Сегмент | Расход | Клики | Реальные лиды | CPL | Квалиф. лиды | CPQL | Встречи | Сделки | Выручка | Маржа | ROMI | Решение |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|---:|---:|---|
| Search / Hot Demand / MSK | | | | | | | | | | | | увеличить / держать / снизить |
| Network / Retargeting / 7-30d | | | | | | | | | | | | увеличить / держать / снизить |
| Network / Cold / Autotargeting | | | | | | | | | | | | тест / ограничить / отключить |

Важный принцип: CTR, CPC и технические конверсии должны быть во вторичном блоке. Они объясняют проблему, но не определяют коммерческое решение.

## 8. Где вероятнее всего реальная проблема

### 8.1. Трафик

Вероятность: высокая.

Причины:

- поиск и сети объединены;
- автотаргетинг потребляет основную часть бюджета;
- 2 853 клика не дали сопоставимого количества подтвержденных лидов;
- внутренние конверсии не подтверждены как бизнес-цели.

Коммерческий вывод: трафик может быть дешевым, но не покупательским.

### 8.2. Оффер

Вероятность: средняя или высокая.

Если аудитория кликает, но не оставляет заявку, оффер может быть недостаточно конкретным:

- непонятно, что именно предлагается;
- нет сильного обещания результата;
- нет понятного следующего шага;
- нет различия между услугами для клиник, стоматологий, SEO, SMM;
- оффер слишком общий для разных сегментов спроса.

Коммерческий вывод: один общий оффер не должен обслуживать все сегменты.

### 8.3. Посадочная страница

Вероятность: высокая.

2853 клика и 1 подтвержденный целевой визит - это сигнал либо о нецелевом трафике, либо о плохой посадочной, либо о сломанной аналитике.

Что проверить:

- мобильная версия;
- скорость загрузки;
- видимость формы и телефона;
- наличие WhatsApp/Telegram;
- соответствие заголовка объявлению;
- наличие кейсов и доказательств;
- понятный CTA;
- корректность отправки формы;
- фиксация звонков и кликов по мессенджерам.

### 8.4. Аналитика

Вероятность: очень высокая.

Конфликт `495 внутренних конверсий` против `1 целевого визита` - это прямой признак, что аналитический контур не готов к управлению прибылью.

Возможные проблемы:

- в Директе считаются автоцели;
- бизнес-цели не подключены к стратегии;
- цели Метрики дублируются или настроены неверно;
- звонки и мессенджеры не передаются в Метрику/CRM;
- CRM не возвращает статус качества лида;
- UTM теряются при переходе.

Коммерческий вывод: без исправления аналитики невозможно понять, реклама плохая или учет лидов плохой.

### 8.5. Отдел продаж

Вероятность: неизвестна, но должна быть проверена.

Если лиды есть, но не подтверждаются, возможны проблемы:

- менеджеры долго отвечают;
- звонки не дозваниваются;
- заявки не попадают в CRM;
- WhatsApp/Telegram не фиксируются;
- лиды помечаются вручную без причины отказа;
- нет SLA первого контакта;
- нет квалификации по источнику.

Коммерческий вывод: performance нельзя оценивать без статусов продаж. Маркетинг может приводить обращения, но бизнес не видит их как лиды.

## 9. Диагноз

Наиболее вероятная корневая проблема - не одна настройка Директа, а разрыв между рекламным кабинетом, аналитикой и CRM.

Реклама генерирует события, но бизнес не может доказать, что эти события становятся лидами и продажами.

Порядок вероятности:

1. Аналитика - очень высокая вероятность проблемы.
2. Трафик - высокая вероятность проблемы.
3. Посадочная страница - высокая вероятность проблемы.
4. Оффер - средняя/высокая вероятность проблемы.
5. Отдел продаж - неизвестно, требует CRM-данных.

## 10. Что делать коммерческому директору

Первые 48 часов:

1. Запретить масштабирование бюджета.
2. Утвердить список бизнес-целей: форма, звонок, запись, WhatsApp, Telegram, подтвержденный лид.
3. Проверить, какие goalId соответствуют этим целям.
4. Проверить, какие цели входят в 495 конверсий.
5. Проверить, попадают ли все формы, звонки и мессенджеры в CRM.
6. Проверить 20-30 последних обращений вручную: источник, качество, статус, менеджер, причина отказа.

Первая неделя:

1. Разделить поиск и сети.
2. Разделить ретаргетинг, автотаргетинг и товарные фильтры.
3. Вынести холодные сети в отдельный тестовый бюджет.
4. Настроить CRM-поля и обязательную передачу UTM/ID.
5. Сделать отчет `расход -> лид -> квалификация -> сделка -> маржа`.
6. Перезапустить оптимизацию только на бизнес-лиды.

После этого:

1. Отключать не сегменты с плохим CTR, а сегменты с плохим CPL/CPQL/CAC.
2. Масштабировать не кампании с дешевыми конверсиями, а сегменты с подтвержденной маржой.
3. Еженедельно сверять Директ, Метрику и CRM.
