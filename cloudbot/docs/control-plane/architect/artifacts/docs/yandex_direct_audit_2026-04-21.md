## 1. Краткий управленческий вывод

Кабинет сейчас держится почти полностью на одной кампании: `Товарная / МК / СПб, Мск`.

За доступный период она дала 487 конверсий при расходе 17 664,99 руб. и среднем CPL 36,27 руб. На уровне общей экономики это выглядит сильным источником лидов, но внутри последних дней есть тревожный тренд: CPC вырос с 2,30 руб. 14 апреля до 35,85 руб. 21 апреля, а CPL вырос с 22,87 руб. до 78,87 руб.

Главная проблема не в том, что кампания не работает. Главная проблема в том, что эффективность быстро деградирует, а структура кабинета не дает достаточной управляемости: активная статистика есть по кампании, но API не вернул детализацию по группам, фразам, поисковым запросам, площадкам, устройствам и регионам. Это ограничивает точечную оптимизацию и указывает на необходимость проверить тип кампании/доступность детализации в интерфейсе Директа.

## 4. Что нужно отключить срочно

Не включать `Ретаргетинг, Неотказы, общая / РСЯ / РФ` в текущем виде.

Причина: CTR 0,04%, CPC 676,81 руб., 0 конверсий. Ретаргетинг должен работать на теплую аудиторию, но здесь нет признаков нормального отклика.

Не масштабировать `Общая / Поиск / Мск СПб` без пересборки.

Причина: CPL 1 556,62 руб. против 36,27 руб. у основной товарной кампании. Поиск может быть качественным каналом, но текущая настройка слишком дорогая.

Не принимать 487 конверсий товарной кампании как финальную бизнес-истину без проверки целей в Метрике.

Причина: конверсия 15,87% при CPL 36,27 руб. выглядит очень хорошо. Нужно подтвердить, что это именно лиды, а не микроцели/автоцели.

## 5. Что переделать в течение недели

Проверить цели Метрики, которые оптимизируют кампании.

В структуре встречаются разные GoalId и автоцель `13`. Нужно разделить:
- настоящие лиды;
- заявки;
- звонки;
- формы;
- клики по мессенджерам;
- автоцели и микроцели.

Для оптимизации оставить только цели, которые коррелируют с реальным лидом. Микроцели не должны управлять стратегией как основные конверсии.

Пересобрать поиск `Общая / Поиск / Мск СПб`.

Нужна новая структура:
- отдельные кампании или группы под транзакционные запросы;
- отдельные группы под SEO, SMM, медицинский маркетинг, стоматологии, клиники;
- отдельный бренд;
- отдельные конкуренты;
- жесткая минусация информационных и нерелевантных интентов;
- объявления под конкретный интент, а не общие тексты.

Проверить посадочные страницы.

Поиск дорогой, поэтому каждая посадочная должна отвечать на конкретный спрос: маркетинг клиники, стоматологии, медицинский SMM, SEO, performance. Если весь трафик ведется на общую страницу, часть дорогих кликов теряется на несоответствии оффера.

Разобрать товарную кампанию по сегментам.

Нужно понять, что именно дает дешевые конверсии:
- товарные предложения;
- аудитории;
- регионы Москва/СПб;
- устройства;
- площадки;
- время;
- конкретные цели.

Сейчас кампания дает результат, но управляемость недостаточная.

## 6. Что требует стратегической переработки

Структура аккаунта.

В кабинете много архивных кампаний и несколько поколений структуры. Это мешает управлению: часть кампаний названа по датам, часть по типам спроса, часть по техническим обозначениям. Нужен единый принцип:

- `Brand / Search / MSK-SPB`;
- `Generic / Search / Clinic Marketing / MSK-SPB`;
- `Generic / Search / Dental Marketing / MSK-SPB`;
- `SMM / Search / MSK-SPB`;
- `SEO / Search / MSK-SPB`;
- `Competitors / Search / MSK`;
- `Retargeting / Network / Warm 7-30d`;
- `Lookalike / Network / Leads / MSK-SPB`;
- `Product / Network / MSK-SPB`.

Стратегии назначения ставок.

В структуре встречаются `WB_MAXIMUM_CLICKS`, `PAY_FOR_CONVERSION_MULTIPLE_GOALS`, `WB_MAXIMUM_CONVERSION_RATE`, `HIGHEST_POSITION`. Это разные логики оптимизации. Нужно привести стратегии к понятной модели:

- поиск по горячим запросам: контроль CPA или ручная стратегия на тесте;
- товарная/сетевая кампания: оптимизация по подтвержденной лидовой цели;
- ретаргетинг: ограниченный бюджет и отдельная цель;
- тестовые кампании: отдельный бюджет, не смешивать с основным лидогенератором.

Система целей.

Сейчас главный риск - оптимизация на цели, которые не равны качественному лиду. Если автоцели или микроцели дают дешевые конверсии, алгоритм может уводить бюджет в трафик без продаж.

## 7. Что даст самый быстрый рост количества лидов

Самый быстрый рост лидов даст аккуратное масштабирование `Товарная / МК / СПб, Мск`, но только после проверки целей.

План:

1. Подтвердить, что конверсии - это реальные лиды.
2. Найти дни/сегменты с CPL до 35 руб.
3. Удерживать дневной бюджет на уровне, где CPC не разгоняется выше 10-15 руб.
4. Не увеличивать бюджет резко: прибавлять 15-20% и смотреть 2-3 дня.
5. Отдельно вынести лучшие связки в управляемые кампании, если интерфейс/тип кампании позволяет.

## 8. Что даст самое быстрое снижение CPL

Самое быстрое снижение CPL даст остановка дорогих источников и защита товарной кампании от разгона CPC.

Приоритет:

1. Не включать ретаргетинг с CTR 0,04%.
2. Не масштабировать поиск с CPL 1 556,62 руб.
3. Проверить цели товарной кампании.
4. Ограничить рост CPC в товарной кампании.
5. Разделить реальные лиды и микроцели.
6. Пересобрать поисковую структуру под горячие интенты.

## 9. Приоритет действий

Высокий приоритет:

1. Проверить цели Метрики и автоцели, которые считаются конверсиями.
2. Зафиксировать, что `Товарная / МК / СПб, Мск` оптимизируется по реальным лидам.
3. Не включать `Ретаргетинг, Неотказы, общая / РСЯ / РФ` без пересборки.
4. Поставить под контроль `Общая / Поиск / Мск СПб`: либо пауза, либо новый тест с жестким лимитом.

Средний приоритет:

1. Пересобрать поиск по интентам.
2. Проверить посадочные страницы под каждую группу спроса.
3. Создать отдельные кампании под бренд, горячий общий спрос, стоматологии, SMM, SEO, конкурентов.
4. Подготовить нормальную карту минус-фраз.

Низкий приоритет:

1. Привести названия архивных и текущих кампаний к единой системе.
2. Документировать правила тестирования новых кампаний.
3. Завести еженедельный отчет по CPL, CPA, CPC, CVR и качеству лидов.
