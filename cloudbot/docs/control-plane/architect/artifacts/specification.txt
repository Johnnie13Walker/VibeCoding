Спецификация приложения Belberry PPC-Auditor

Дата: 21 апреля 2026 г.
Часовой пояс: Europe/Moscow

1. Назначение приложения

Belberry PPC-Auditor предназначен для анализа рекламных кампаний Яндекс.Директа через API и подготовки performance-аудита рекламного кабинета.

Приложение помогает пользователю понять:
- где теряется рекламный бюджет;
- какие кампании, группы объявлений, объявления и ключевые фразы работают неэффективно;
- какие поисковые запросы нужно добавить в минус-фразы;
- какие площадки РСЯ требуют отключения или дополнительной проверки;
- где есть проблемы со ставками, стратегиями, расписаниями и корректировками;
- где можно быстрее всего увеличить количество лидов;
- где можно быстрее всего снизить CPL и CPA.

На первом этапе приложение работает только в read-only режиме. Оно не изменяет кампании, объявления, ставки, минус-фразы, стратегии или другие настройки рекламного кабинета без отдельного подтверждения пользователя.

2. Какие новые возможности приложение дает пользователю

Приложение дает пользователю возможность быстро получать независимый performance-аудит рекламного кабинета Яндекс.Директа на основе данных API.

Пользователь сможет анализировать:
- кампании;
- группы объявлений;
- объявления;
- ключевые фразы;
- поисковые запросы;
- минус-фразы;
- расходы;
- показы;
- клики;
- CTR;
- CPC;
- конверсии;
- CPL;
- CPA;
- площадки РСЯ;
- устройства;
- регионы;
- время показа;
- автоцели;
- стратегии назначения ставок;
- корректировки ставок;
- расписания показов;
- качество структуры аккаунта.

Результатом работы приложения является отчет с управленческими выводами и конкретными рекомендациями:
- что отключить срочно;
- что переделать в течение недели;
- что требует стратегической переработки;
- что даст самый быстрый рост количества лидов;
- что даст самое быстрое снижение CPL;
- какие действия имеют наибольший приоритет по влиянию на результат.

3. Схема взаимодействия приложения с Яндекс.Директом

1. Пользователь авторизует приложение через OAuth Яндекса и предоставляет разрешение direct:api.
2. Приложение получает OAuth-токен и использует его для read-only запросов к JSON API Яндекс.Директа.
3. Через JSON API приложение получает данные о структуре рекламного кабинета:
   - кампаниях;
   - группах объявлений;
   - объявлениях;
   - ключевых фразах;
   - минус-фразах;
   - стратегиях;
   - расписаниях;
   - корректировках ставок;
   - регионах показов;
   - параметрах кампаний и групп.
4. Через Reports API приложение получает статистику:
   - расходы;
   - показы;
   - клики;
   - CTR;
   - CPC;
   - конверсии;
   - CPL;
   - CPA;
   - поисковые запросы;
   - площадки РСЯ;
   - устройства;
   - регионы;
   - время показа.
5. Полученные данные обрабатываются внутри приложения.
6. Приложение формирует отчет для пользователя.
7. Пользователь получает отчет с выводами, приоритетами и рекомендациями.
8. На первом этапе приложение не выполняет автоматическое изменение кампаний, ставок, объявлений или минус-фраз. Любые изменения могут применяться только после отдельного подтверждения пользователя.

4. Используемые методы API

Планируемое использование API:
- Clients.get или AgencyClients.get для определения доступного рекламного кабинета и необходимости Client-Login;
- Campaigns.get для получения кампаний и настроек;
- AdGroups.get для получения групп объявлений;
- Ads.get для получения объявлений;
- Keywords.get для получения ключевых фраз;
- Bids.get для анализа ставок;
- Dictionaries.get для справочных данных;
- Reports API для получения статистики по кампаниям, фразам, поисковым запросам, площадкам, устройствам, регионам и времени показа.

Методы изменения данных на первом этапе не используются.

5. Безопасность и ограничения

Приложение не хранит API-ключи, OAuth-токены и другие секреты в публичном репозитории.

OAuth-токен используется только для запросов к API Яндекс.Директа от имени авторизованного пользователя.

На первом этапе приложение работает как аналитический инструмент:
- не создает кампании;
- не редактирует кампании;
- не изменяет ставки;
- не добавляет и не удаляет ключевые фразы;
- не добавляет и не удаляет минус-фразы;
- не изменяет объявления;
- не меняет стратегии;
- не меняет расписания показов.

Все рекомендации сначала формируются в отчете. Применение изменений возможно только после отдельного подтверждения пользователя.

6. Формат результата для пользователя

Итоговый отчет содержит:

1. Краткий управленческий вывод.
2. Основные точки потери денег.
3. Что нужно отключить срочно.
4. Что нужно переделать в течение недели.
5. Что требует стратегической переработки.
6. Что даст самый быстрый рост количества лидов.
7. Что даст самое быстрое снижение CPL.
8. Приоритет действий по степени влияния на результат.

Отчет не ограничивается перечислением метрик. Основная задача отчета - дать пользователю конкретные управленческие выводы и действия для повышения эффективности рекламного кабинета.
