# DevOps / SRE регламент

## Назначение
Эта папка описывает функцию инженера, который отвечает за стабильность серверов, порядок в инфраструктуре и регулярные проверки работоспособности сервисов.

Роль в команде:
- DevOps Engineer: инфраструктура, CI/CD, деплои, автоматизация.
- SRE: надежность, мониторинг, алерты, SLI/SLO, инциденты.

## Зона ответственности
- Доступность сервисов и API.
- Стабильность деплоев и откатов.
- Мониторинг, алертинг, логирование.
- Резервное копирование и восстановление.
- Безопасность инфраструктуры (обновления, ключи, доступы).
- Документация и runbook-процедуры.

## Регулярные проверки

## Ежедневно
- Проверить, что все критичные сервисы `active/running`.
- Проверить ошибки в логах за последние 24 часа.
- Проверить алерты (критичные/major), закрыть или эскалировать.
- Проверить свободное место на дисках и память.
- Проверить успешность ночных backup-задач.

## Еженедельно
- Проверить время ответа и стабильность ключевых эндпоинтов.
- Проверить свежесть SSL-сертификатов.
- Проверить состояние очередей/кронов/воркеров.
- Проверить отложенные инциденты и технический долг.
- Протестировать один сценарий восстановления (минимальный DR-check).

## Ежемесячно
- Обновить пакеты безопасности и критичные зависимости.
- Провести аудит прав доступа (SSH, токены, сервисные аккаунты).
- Проверить и обновить документацию по инфраструктуре.
- Сверить фактическую конфигурацию с desired state.
- Провести postmortem по инцидентам за месяц и зафиксировать улучшения.

## Базовый runbook инцидента
1. Подтвердить инцидент (факт, время начала, затронутые сервисы).
2. Снизить ущерб (rollback, restart, переключение на fallback).
3. Зафиксировать статус в канале инцидента.
4. Найти первопричину по логам/метрикам/трейсам.
5. Выпустить фикс и проверить восстановление.
6. Оформить postmortem: причина, влияние, действия, prevention.

## Минимальный чеклист перед прод-деплоем
- Пройдены тесты CI.
- Есть план отката.
- Есть backup критичных данных.
- Мониторинг и алерты включены.
- Ответственный инженер на связи.

## Быстрый старт
1. Перейти в папку:
   - `cd /Users/pro2kuror/Desktop/OpenClo/incubator/openclaw-extensions/devops-sre`
2. Сделать скрипты исполняемыми:
   - `chmod +x checks/*.sh`
3. Запустить проверку вручную:
   - `./checks/health_check.sh --services "nginx docker"`
4. Запустить ежедневный чек-лист:
   - `./checks/daily_check.sh --services "nginx docker"`
5. Запустить еженедельный аудит:
   - `./checks/weekly_check.sh --services "nginx docker"`
6. Запустить полный набор проверок:
   - `./checks/run_all.sh --services "nginx docker"`

## Подключение в systemd (на сервере)
1. Скопировать файлы из `systemd/` в `/etc/systemd/system/`.
2. Проверить пути внутри `ExecStart` (должны указывать на ваш абсолютный путь).
3. Выполнить:
   - `sudo systemctl daemon-reload`
   - `sudo systemctl enable --now devops-healthcheck.timer`
   - `sudo systemctl enable --now devops-weekly-audit.timer`
4. Проверить статус:
   - `systemctl status devops-healthcheck.timer`
   - `systemctl status devops-weekly-audit.timer`

## Telegram уведомления
Скрипты daily/weekly отправляют уведомление в Telegram после успешного запуска.

Настройка:
1. Создать файл `/opt/devops-sre/.env` на сервере:
   - `TELEGRAM_BOT_TOKEN=...`
   - `TELEGRAM_CHAT_ID=...`
2. Или использовать fallback из `/opt/projects/.autopilot.env` (если уже настроен).

## Мониторинг промокодов Победы (Telegram)
Добавлен watcher, который проверяет публичный канал `@PobedaAirlines` и присылает уведомление, если в новых постах найден промокод.

Что используется:
- Скрипт: `checks/pobeda_promocode_watch.py`
- Service: `systemd/pobeda-promocode-watch.service`
- Timer: `systemd/pobeda-promocode-watch.timer` (каждые 15 минут)

Ручной запуск:
- `python3 checks/pobeda_promocode_watch.py --channel PobedaAirlines`

Важно:
- При первом запуске watcher делает bootstrap: запоминает текущий последний пост и не шлет старые уведомления.
- Состояние хранится в `logs/pobeda_promocode_state.json`.
- Канал можно переопределить через env `PROMO_CHANNEL`.

Включение через systemd:
1. Установить/обновить юниты:
   - `sudo ./scripts/install_systemd.sh "/absolute/path/to/project"`
2. Проверить:
   - `systemctl status pobeda-promocode-watch.timer`
   - `systemctl list-timers | grep pobeda-promocode-watch`
