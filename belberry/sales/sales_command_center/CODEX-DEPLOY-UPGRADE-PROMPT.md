**5. Перегенерация двух последних рабочих дней новым кодом — БЕЗ Telegram.**
Петля «обещали→выполнили» на дне N читает структурные `next_step` дня N−1, поэтому регенерим ДВА дня по порядку (как валидировалось на preview):
```bash
cd runner
# Telegram заглушаем на время ручной регенерации:
TELEGRAM_CHAT_ID= TELEGRAM_BOT_TOKEN= .venv/bin/python daily_runner.py --date <предпоследний раб.день> --force
TELEGRAM_CHAT_ID= TELEGRAM_BOT_TOKEN= .venv/bin/python daily_runner.py --date <последний раб.день> --force
```
Подставить реальные даты (например, если выкатка 04.06 — это 02.06 и 03.06). Проверить в выводе/БД: `summary_json->>'llm_status' = 'done'`, ошибок нет.

**6. Смоук в вебе (визуально).**
Залогиниться, открыть `/day/<последний раб.день>` и убедиться, что виден НОВЫЙ формат:
- hero-stats плитки + Δ к прошлому дню, число health-score рядом;
- якорная навигация (Главное·Тигр·Встречи·Разбор·Риски);
- разбор встреч: цитата клиента + hilight-сетка + структурный «следующий шаг» (где есть транскрипт); бейдж «разбор по краткому статусу, не по транскрипту» — где транскрипта нет;
- секции «ТМ-воронка» и «Вчера обещали → сегодня» (петля; на первом дне может быть «нет структурных обещаний» — это норма для перехода);
- риск-теги (нет бюджета / молчит N дн / застрял / нет контакта), data-quality в подвале.
Проверить мобильную ширину (~390px: карточки стекаются, таблицы скроллятся) и print-preview.

## Заметки / возможные правки (не блокеры, на усмотрение)
- **Модель LLM.** Preview апгрейда валидирован на `gpt-5.5`. Новый SYSTEM_PROMPT крупный и требовательный. Проверь `LLM_MODEL` в `/etc/scc/scc.env`: если там `gpt-4o`, реши с пользователем, переключать ли на `gpt-5.5` для паритета (одна строка env, без правок кода). Сообщи, что стоит.
- **Health-score** упирается в штраф −25 в дни с ~8+ зависшими сделками (наблюдалось на preview 06-02: score 40). Это известный вопрос калибровки весов с РОПом — не блокер выкатки.
- **Метка `skipped_no_transcript`** ставится и когда транскрипт был, но LLM вернул невалидный JSON (тогда `transcript_based=true`). Бейдж честности корректен, метка в БД неточна — мелочь на будущее.
