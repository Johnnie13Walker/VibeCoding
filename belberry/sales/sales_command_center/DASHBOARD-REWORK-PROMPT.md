## Что за проект
Веб-приложение для отдела продаж Belberry: Python-раннер каждое утро собирает данные из Bitrix24
(сделки, телефония, встречи с LLM-разбором транскриптов, брифы, КП) → пишет в PostgreSQL →
Next.js читает и показывает: дневной отчёт (`/daily`), live «Сегодня» (`/today`), алерты (`/alerts`),
**дашборд отдела (`/dashboard`)**. Пользователи — руководитель (Щемелёв), РОП, менеджеры.

- **KPI-карточки** (`KpiCard`): сумма воронки, встречи/наборы/сделки за период + дельта к прошлому периоду + спарклайн-тренд.
- **Воронка продаж** (`FunnelBars`) по стадиям `STAGE_META` (категория 10), снимок последнего дня.
- **Застрявшие сделки** (`stuck`).
- **Команда** (`TeamList`) — по менеджерам (встречи назначено/проведено, наборы, КП, брифы, часы разговоров, тренд), клик по строке → дрилл-даун с последними разобранными встречами (балл+заметка).
- **План/факт встреч** (`Goal`) — цель из таблицы `plans` (план встреч на человека).
- Переключатель Месяц/Неделя; кнопка на свежий дневной отчёт.

Известный нюанс/блокер: **источник плановых цифр** (таблица `plans`) — план встреч/чел; план-факт по деньгам/продуктам пока не заведён.

## КЛЮЧЕВЫЕ ПРАВИЛА ПРОЕКТА (соблюдать в любой метрике/тексте)
- Язык: **русский** (UI, комментарии, коммиты). **Без англицизмов** в интерфейсе (дозвон, не «коннект»; сделка, не «deal»; воронка, не «пайплайн»).
- Время — **Europe/Moscow** везде.
- **Дозвон = разговор ≥60 секунд** (`calls_60s_plus`), а не «снял трубку» (`calls_answered`).
- **Встреча «назначено» засчитывается СОЗДАТЕЛЮ** (телемаркетологу), проведённая — ответственному. В `manager_activity.meetings_set` уже по создателю.
- **Скоуп — только отдел продаж + телемаркетинг** (`isSalesDept` в `lib/dashboard.ts`); HR/аккаунтинг/дизайн исключать.
- **СПАМ-сделки** (`UF_CRM_1771495464 = 8588`) исключать из новых сделок/отказов.
- Drizzle: timestamptz без `mode:'date'` может прийти строкой — нормализуй к Date (см. `mapRow` в lib).
