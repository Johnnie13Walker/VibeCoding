## Главный вывод за 30 секунд

Год SEO дал реальный рост трафика, но не сформировал новый коммерческий спрос. Претензия не в том, что «ничего не сделали»: Метрика показывает рост визитов, органики и форм. Претензия в том, что рост собран в основном на бренде, информационных запросах и технически слабой Tilda-структуре, а не на контролируемой системе привлечения платящих B2B-клиентов.

1. **Трафик вырос, но спрос не тот.** За год визиты выросли с 624 до 1802 в месяц, органика — с 233 до 825, формы — с 10 до 36 в месяц. По году: 15 578 визитов, 7 116 органика (46%), 276 форм. Но топ-фразы — бренд и информационный КЭР, а не «купить/внедрить СППВР».
2. **Лид есть, договоров мало.** По словам клиента, из ~35 заявок до договора доходят около 2, то есть примерно 6%. При среднем чеке 300–400 тыс. ₽ оптимизация должна считаться в целевых заявках и договорах, а не в визитах.
3. **Предыдущий подрядчик оптимизировал отчётность, а не качество лида.** Вторая по входам страница — `/blog/clinicalexpertwork` (222 визита за месяц), а топ-фраза — «организация кэр в лпу». Это хорошо для видимости, но слабо связано с покупкой SaaS.
4. **Tilda уже показывает потолок.** Есть canonical и sitemap, но нет JSON-LD, массово пустые `alt`, H1 «ИИ» и «ДМС» на коммерческих страницах, мусорные старые URL в sitemap, ограниченный контроль шаблонов, скорости, редиректов и масштабируемой структуры.
5. **Сквозная аналитика не собрана.** Есть Яндекс.Метрика, но нет GA4/GTM, коллтрекинга и связки «источник → заявка → демо → договор». Значит, SEO нельзя оптимизировать на платящий лид.
6. **Стратегический вывод сохраняется.** Краткосрочно Tilda можно использовать для быстрых тестов спроса. Долгосрочно медицинскому SaaS нужен полноценный сайт на 1С-Битрикс или сопоставимой CMS: управляемые шаблоны, микроразметка, контентная структура, аналитика, интеграции, личный кабинет и биллинг.

## 3. Почему Tilda создаёт потолок для SEO и маркетинга

Tilda не является «плохой» платформой. Она хорошо подходит для быстрого лендинга и проверки спроса. Но текущая задача «Панацеи» — не просто лендинг: это системный B2B SaaS-маркетинг с несколькими продуктами, длинной сделкой, сложным поисковым спросом и необходимостью аналитики до договора.

| Что нужно Панацее | Что видно на Tilda сейчас | Риск |
|---|---|---|
| Управляемые шаблоны метаданных | Метаданные есть, но правятся постранично и легко расходятся по качеству | Нельзя масштабировать SEO без ручной рутины |
| Чистая структура блога | Sitemap содержит старые и служебные URL, часть ссылок ведёт через редиректы | Сканирование тратится на мусор |
| Микроразметка | JSON-LD не найден на 8/8 ключевых URL | Слабая понятность для поисковиков и ИИ-ответов |
| Масштабируемые посадочные | Нет нормальной структуры кластеров «продукт × сегмент × боль» | Коммерческое ядро остаётся без посадочных |
| Скорость и контроль ресурсов | HTML главной 2,08 МБ, много Tilda-скриптов, все ключевые страницы тяжёлые | Core Web Vitals лечатся ограниченно |
| Сквозная аналитика | Метрика есть, GA4/GTM/коллтрекинг не найдены в HTML | Нельзя оптимизировать на договор |
| Продуктовый сценарий | Личный кабинет, триал, биллинг, счёт и CRM-интеграции не являются сильной стороной Tilda | Сайт не становится SaaS-инструментом продаж |

**Вывод:** на Tilda можно подкрутить title, H1, alt, ссылки, sitemap и тексты. Но это косметика вокруг платформенного ограничения. Долгосрочно нужен сайт, где SEO-логика живёт в шаблонах, а не в ручной сборке страниц.

## 8. Ссылочный профиль и риск просадки после подрядчика

### 8.1. Что известно

| Метрика | Значение | Источник |
|---|---:|---|
| Бэклинки | 40 | исходный аудит / данные клиента |
| Ссылающиеся домены | 11 | исходный аудит / данные клиента |
| Trust | низкий | исходный аудит / данные клиента |
| GA4 / коллтрекинг / сквозная аналитика | нет | HTML и данные клиента |

Полного списка доноров, URL-акцепторов, анкоров, типов ссылок и дат появления нет. Поэтому детальный Disavow-лист сейчас составлять нельзя: это был бы риск навредить.

### 8.2. Что можно сказать уже сейчас

| Вопрос | Ответ |
|---|---|
| Это сильный ссылочный профиль? | Нет. 11 доменов для года закупки ссылок — слабый результат для B2B SaaS. |
| Есть ли риск просадки после ухода подрядчика? | Катастрофическая просадка маловероятна: текущая органика в основном бренд + информационный КЭР, а не конкурентные коммерческие запросы. |
| Что может просесть? | Часть небрендовой информационки, если ссылки арендные и поддерживали отдельные статьи. |
| Нужно ли срочно всё отклонять через Disavow? | Нет. Без списка доноров и признаков токсичности Disavow делать нельзя. |
| Что важно забрать у подрядчика? | Список купленных ссылок, тип размещения, срок оплаты, аккаунты бирж/кабинетов, договорённости по вечным публикациям. |

### 8.3. Таблица разбора ссылок: текущий статус

| Параметр | Статус | Что нужно для подтверждения | Решение |
|---|---|---|---|
| Типы доноров: СМИ, каталоги, форумы, биржи, статейники | Needs data | Экспорт Ahrefs/Serpstat/Яндекс.Вебмастер/Google Search Console | Классифицировать доноров по качеству |
| Анкоры | Needs data | Список anchor text | Проверить переспам «СППВР», «ИИ», бренд, URL-анкоры |
| Токсичность | Needs data | DR/ИКС/спамность/тематика/индексация доноров | Disavow только для явно токсичных или взломанных доноров |
| Аренда vs вечные | Needs data | Договор/кабинеты подрядчика/биржи | Перенести вечные в собственность клиента; арендные отключать постепенно |
| Ссылки на старые URL | Likely issue | Экспорт входящих ссылок по целевым URL | Поставить 301 со старых URL на новые, чтобы не терять вес |

### 8.4. План безопасной работы со ссылками

| Шаг | Что сделать | Acceptance criteria | Владелец |
|---|---|---|---|
| 1 | Получить выгрузку всех ссылок из Яндекс.Вебмастера, GSC и внешнего сервиса | В таблице есть donor URL, target URL, anchor, first seen, last seen, type, status | SEO |
| 2 | Разделить ссылки на owned / earned / paid eternal / paid rental / toxic / unknown | У каждой ссылки есть класс и решение | SEO |
| 3 | Проверить, ведут ли ссылки на `*-old`, `/page*.html`, старые пути | Все ссылочные старые URL имеют корректный 301 на актуальный URL | Dev / SEO |
| 4 | Не отключать арендные ссылки залпом | Отключение только после оценки трафика/позиций и группами | SEO |
| 5 | Сформировать Disavow только после ручной проверки | В Disavow попадают только явно токсичные домены; есть обоснование по каждому | SEO |
| 6 | Начать собственный PR/link earning | 2–4 качественных отраслевых упоминания в месяц: медтех, ИТ в медицине, кейсы клиник | PR / SEO |
