## 3. Сводка-инсайты

### 3.1. Общие закономерности выигравших КП
1. **Главный выигрышный формат — прозрачная постатейная смета, а не презентация.** 13 из 19 сделок закрыты таблицей: работы → часы → деньги → итог. Презентации (3 шт.) встречаются только в SEO и контекстной рекламе для «холодных» клиентов.
2. **Цены всегда без НДС** (УСН), с явной пометкой «БЕЗ НДС» в итогах. Исключение — рекламные бюджеты в медиапланах: там «с НДС», потому что это деньги Яндекса.
3. **Скидки — система из трёх рычагов, повторяется почти везде:**
   - 10 % за размещение копирайта/логотипа агентства (вся разработка сайтов; закреплена и в договоре с условием возврата при удалении);
   - 5 % «за оперативное решение» с явным сроком (10 дней / «при оплате до даты»);
   - лестница за предоплату: 3/6/9/12 месяцев = 5–7 / 10 / 15 / 20 %.
4. **Каскад итоговых цен** — «полная → со скидкой 1 → со скидкой 2» — стандартный приём: клиент видит выгоду в рублях, а не в процентах.
5. **Часы как валюта доверия.** Почти каждая строка сметы имеет колонку «КОЛ-ВО часов»; в SEO бюджет подаётся как «N ₽ / M часов в месяц». Единая ставка ~2 500–2 800 ₽/час просматривается насквозь.
6. **Опции нулями.** Невыбранные блоки (этап на VUE, программирование) показываются в смете с нулевой ценой — расширяют картину возможного без увеличения чека.
7. **Этапность и поэтапная оплата:** Дизайн → Вёрстка → Программирование и наполнение; оплата траншами до начала каждого этапа.
8. **Прогнозы — только с оговорками.** «Медиаплан… является примерным», три сценария трафика («гарантированный» = реалистичный), честный отказ прогнозировать лиды, когда нет данных, с перечислением причин.
9. **Срок действия КП** на титуле презентаций («Актуально до …») — встроенный стимул решать быстро.
10. **Персонализация:** имя ответственного менеджера на первом и последнем слайде; персональная аналитика сайта клиента (его трафик, его ошибки, его конкуренты) — самая сильная часть презентаций.
11. **Допродажа внутри КП:** блок «Ещё больше возможностей» с ценами «от 50–65 000 ₽/мес» (презентации) или лист «Техническая поддержка» (сметы разработки).

### 3.2. Belberry против Acoola Team
- Шаблоны общие: одна и та же смета разработки с переключателем «скидка за копирайт BELBERRY/ACOOLA TEAM», один прайс ORM на обе марки. Бренд — это надпись, а не отдельная структура КП.
- Различие в содержании, а не в форме: у Belberry — отраслевой шаблон «WD Med» (страницы Врачи, Программы, Анализы, интеграция с МИС, аудит EEAT «для мед. сайтов»), у Acoola — «WD E-com»/корпоративный и универсальные презентации.
- В выборке большие презентации — у Acoola (более холодные, немедицинские клиенты); Belberry чаще выигрывает короткой сметой — медицинский клиент уже прогрет встречей.

### 3.3. Различия по услугам
- **WD (разработка сайтов):** постраничная смета по 3 этапам, исполнители по ролям, лицензия Битрикс строкой, скидка 10 % за копирайт, допродажа техподдержки. Без текста-продажи вообще.
- **SEO:** два формата. Холодному клиенту — презентация 40–54 слайда с персональной аналитикой, тремя сценариями прогноза и примерами работ; тёплому — рабочая книга «план работ на 24 недели + часы/деньги». Цена — помесячная, «N ₽ / M часов».
- **PPC (контекст):** медиаплан с прогнозом до уровня CPL (иногда по каждому запросу), таблица работ с «+», цена «фикс + 10 % от бюджета» либо фикс при ограничении бюджета, дисклеймер про прогнозы Директа.
- **ORM:** пакеты S/M/L с «минимальным чеком», конструктор объёмов (отзывы поштучно, «включено» для стратегии/отчётности), лестница скидок за предоплату.
- **SMM:** помесячная смета из 4 блоков (Аналитика / Контент / Реклама / Менеджмент), количества единиц контента, «Рекламный бюджет оплачивается отдельно».
- **LP (посадочная страница):** одна страница, 7 строк, каскад скидок с дедлайном.
- **Program (контент/тексты):** продаётся демонстрацией структуры (интеллект-карта с семантикой) либо строками внутри SEO-сметы.

### 3.4. Чего в выигравших КП НЕТ (движку не усложнять)
- Нет KPI-гарантий и обязательств по результату — даже слово «гарантированный» применяется только к названию сценария прогноза с расшифровкой «реалистичный».
- Нет маркетингового текста в сметах: ни «о компании», ни миссии, ни преимуществ — таблица и итог.
- Нет примеров работ в сметах (только в презентациях).
- Нет расчётов окупаемости/возврата вложений ни в одном из 19 файлов.
- Нет сроков исполнения в большинстве КП (сроки появляются только в договоре).
- Нет вилок «от … до» — везде конкретные суммы.
- Антипаттерн (повторился 4 раза): клиенту уходят внутренние листы — «Калькулятор ЛЕРА», листы исполнителей «Ксюша/Андрей/Дима», пометки «Эта часть внутренняя», общий прайс-сборник целиком, чужие домены в недочищенном шаблоне (waterproof.ru в КП для defagroup). Сделки всё равно выиграны, но это риск: видны себестоимость и внутренние цены.

## 5. Рекомендации движку КП

### Обязательные блоки (везде)
1. **Смета-таблица**: наименование работы → стоимость → количество → единица (часы/штуки) → исполнитель (для разработки). Это ядро; всё остальное — обвязка.
2. **Итог «БЕЗ НДС»** + каскад скидок в рублях (полная цена → −10 % копирайт → −5 % оперативность с датой).
3. **Лестница скидок за предоплату** для помесячных услуг: 3/6/9/12 мес = 5(7)/10/15/20 %.
4. **Оговорка при любом прогнозе** («является примерным», «реалистичный/оптимистичный сценарий»). Если данных для прогноза нет — писать об этом прямо с причинами, это выигрывало.
5. **Срок действия КП** («Актуально до» = +30 дней).
6. **Имя ответственного менеджера** с контактами.

### Опциональные блоки (по контексту)
- **Презентационная обвязка** (факты об агентстве, сертификаты, ликбез, аналитика сайта клиента, стратегия, команда, примеры работ, допуслуги) — только для SEO/PPC и холодных клиентов. Для тёплых после встречи — смета без обвязки.
- **Примеры работ** — только в презентациях; формат: ниша + период + «в N раз / с X до Y ₽ цена лида».
- **Допродажа**: лист «Техническая поддержка» в сметах разработки; «Ещё больше возможностей, от N ₽/мес» в презентациях.
- **Опции нулями** (этап на VUE, доинтеграции) — показывать в смете разработки.

### Состав по услугам
| Услуга | Формат | Особенности цены |
|---|---|---|
| WD | Смета: 3 этапа, постранично, роли | Без НДС, скидка 10 % копирайт, лицензия Битрикс строкой, поэтапная оплата |
| SEO | Презентация (холодный) или план работ на 24 недели (тёплый) | «N ₽ / M часов в месяц», предоплатная лестница |
| PPC | Медиаплан до CPL + таблица работ с «+» | «Фикс + 10 % от бюджета», бюджет с НДС отдельно |
| ORM | Пакеты S/M/L либо конструктор объёмов | Минимальный чек, предоплатная лестница |
| SMM | Помесячная смета из 4 блоков | Бюджет на рекламу отдельно, «Включено» для менеджмента |
| LP | Одностраничная смета 5–8 строк | Каскад скидок с дедлайном |
| Program | Строки в SEO-смете или демонстрация структуры контента | Поштучно (ТЗ, статьи) |

### Чего движку НЕ делать
- Не добавлять KPI-обязательства, расчёты окупаемости, длинные продающие тексты в сметы — в выигравших КП этого нет.
- Не использовать вилки цен — только конкретные суммы.
- **Жёстко разделять клиентскую и внутреннюю части**: генерировать только клиентские листы. В 4 из 19 выигравших файлов клиенту утекли внутренние калькуляторы, листы исполнителей и пометки «это внутреннее» — движок должен закрыть эту дыру по построению.
- Чистить шаблонные хвосты: проверка, что в тексте нет доменов других клиентов (прецедент — waterproof.ru и packing-group.ru в КП для defagroup.com).

- 9598_rudenta.ru — интеллект-карта структуры контента, а не КП.
