## 0. Решения (2026-06-10, обновлено после сбора данных)

1. **MVP стартует с SEO-Belberry** — эталон med-shushary + готовые авто-аудиты, сквозной проход первым.
2. **НДС: все сделки формируем с 5% без указания юрлица.** Юрлицо ООО «Акула Тим» (22%)
   пока не продаёт. Поле выбора юрлица в UI на MVP НЕ нужно — фикс ×1.05.
3. **Кейсы можно публиковать все** (подтверждено заказчиком) — флаг «можно_в_КП» не нужен;
   для MVP берём активные 2023+ с заполненными ссылками из портфолио-таблицы.
4. **KPI с клиентами НЕТ.** Блок «Гарантии/KPI» в шаблонах заменить на «Наши обязательства»
   (процессные: объём работ, отчётность, прозрачность) — НЕ обещания результата.
5. **ToV единый принцип:** экспертная подача, но понятным клиенту языком; минимум англицизмов.
6. **Стиль Acoola-шаблона** (с сайта acoola.team, снято 10.06): фирменный синий `#3086FB`,
   тёмно-серый текст `#313131` (НЕ чистый чёрный — требование «не борщить с чёрным»),
   светлые фоны `#F2F2F4`/белый, акцент-красный `#ff4545` точечно; заголовки — широкий
   гротеск (Druk Text Wide Cy), текст Gilroy/Manrope.
7. **Метрика:** доступ по Belberry есть; по Acoola — дадут позже.
8. **Страхи/боли клиентов по услугам** — формирую сам от типовых (вопрос 8 заказчика),
   валидируются по транскриптам реальных встреч (deal_brain).

  (файл UF_CRM_1666175611166 / ссылка UF_CRM_1739894236555) — для авто-подтяжки и

**Законодательство:** готово — [LEGAL-MED-2026.md](LEGAL-MED-2026.md) (10 разделов, всё

  перепроверить заменяющий акт).
- Ст. 24 ФЗ-38: предупреждение «имеются противопоказания…» ≥5% площади в интернет-рекламе;
  запрет случаев излечения и благодарностей пациентов в рекламе; реклама абортов запрещена.
- ФЗ-152: с 01.09.2025 согласие на ПД — строго отдельным документом; утечки «здоровья» —
  штраф 10–15 млн ₽, повторно оборотные 1–3%.
- Маркировка: erid обязателен, сбор 3% с интернет-рекламы; запрет рекламы в Instagram/
  Facebook (ФЗ-72), посевы — только каналы из реестра РКН.
- ORM-критично: факт обращения пациента = **врачебная тайна** — в ответах на отзывы клиника
  не подтверждает обращение; накрутка отзывов = недобросовестная конкуренция (100–500 тыс. ₽).
- Телемедицина: диагноз только очно — в КП не обещать «лечение не выходя из дома».
- ГОСТ Р 52872-2019 добровольный, но версию для слабовидящих делает обязательной 956н.

## 4. Сценарий сейлса

1. Карточка встречи / `/kp <deal>` → предзаполнение (бренд по отрасли, домен, бриф,
   транскрипты, конкуренты из брифа).
2. Выбор: бренд → тип(ы) → услуги → ниша.
3. «Запустить анализ» → пайплайн (аудиты+Метрика+Bitrix+кейсы), прогресс по стадиям.
4. Черновик: слайды + аргументация + смета; блоки помечены ✅факт/⚠проверить/✋ввод.
5. Редактор: правка текста/цен/кейсов, авто-пересчёт НДС+скидок.
6. QA-проверка (анти-галлюцинация) → список «требует проверки».
7. Экспорт PDF/PPTX/DOCX/Google Docs.
8. Прикреп в Bitrix по кнопке (.docx→поле «Аудит сделки», .pdf→таймлайн).
9. Версия в kp_jobs.

## 6. Подбор кейсов

Фильтры по весу: бренд → ниша → услуга → тип задачи → размер → проблема → есть_цифры → можно_в_КП.
Скоринг: ниша+услуга=100; одно из=70; смежная=40.
Fallback: точный → близкий(плашка «смежный», для Belberry обяз.) → общий по услуге →
«опыт команды» без выдуманных цифр.
Блокер: поля «можно_в_КП»/«цифры» в таблице не структурированы → разметка (этап 2).

## 10. Анти-галлюцинации

Каждая цифра — с источником, иначе не в КП. Кейсы только из таблицы с можно_в_КП=да.
Цифры — только из audit/metrika/bitrix/prodoctorov.json. Прогнозы — диапазоны+дисклеймер.
Никаких «гарантируем топ». Слой проверки размечает: ✅факт / ⚠гипотеза / ✋ввод / ⛔без
источника(блок экспорта). Цены не выдумывать — нет сметы → запрос сметчику. Мин.чек —
нельзя опустить ниже минимума направления.

## 11. Этапы

1. **MVP** — /kp в SCC, выбор бренд+тип, ручной ввод, сборка из 2 эталонов, PDF, прикреп, kp_jobs.

5. **Обучение** — обратная связь, привязка к статусу сделки, библиотека формулировок.
