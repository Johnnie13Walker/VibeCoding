Что изменилось визуально — без потери информации

  • NEWTOC в шапке — строка 📊 Итоги · 📞 Команда · 📈 Воронка · 🤝 Встречи … сразу даёт карту сообщения, ориентир для глаза.
  • CHGВсе цифровые блоки → <pre> с моноширинной таблицей. Вместо «• Активные сделки: 66 / 7 704 041 ₽ (-2 к прошлому дню)» — выровненные колонки факт / ₽ / Δ. Скан глазом за 1 секунду вместо 5 секунд парсинга bullet'а.
  • CHGДеньги в компактном формате: 7.7M ₽ вместо 7 704 041 ₽. Длинные числа разрывают сетку.
  • NEWДельта-стрелки ▲ ▼ ▬ вместо «(-2 к прошлому дню)». Двукратно короче и читается быстрее.
  • NEWЭмодзи-статусы перед каждой строкой риска: 🟠/🟡/🔴 = severity. Цветной маркер слева — стандартный приём дашбордов.
  • CHGSM и TM в одном <pre>, но с разделителем-рядом ──── SM ──── / ──── TM ────. Сравнения не возникает, нормы у них разные.
  • NEW<blockquote> для группировки — список сделок «На стадии договора», риск-сделки, action-list. Telegram рендерит вертикальной полоской слева → визуально отделяется от заголовка как карточка.
  • NEW<blockquote expandable> для длинных хвостов. В превью симулируется fade-out + «▼ показать полностью». В Telegram (Bot API 7.0+) — реально складывается, разворачивается тапом. Так в одно сообщение влезает ВСЕ 9 + 11 + 3 сделки, но видны только top-N.
  • CHGПустые секции схлопнуты в одну строку: 🆕 Новые сделки · нет вместо двухстрочной заглушки. 5 заглушек → 5 строк вместо 15.
  • CHGAction-list в Фокусе обёрнут в <blockquote> — звонки и провалы стали визуальными «карточками». Контроль вынесен отдельно, время подчёркнуто (<u>): 13:00, конца дня.
  • NEWГоризонтальные разделители ━━━━━━ между логическими блоками вместо пустых строк. Telegram сохраняет полную ширину Unicode-линии — она работает как hr.
  • CHG<code> для денежных сумм в детальных строках: 263k ₽ с моноширинным фоном — числа выделяются на фоне текста.

Все изменения — только в файле sales_formatter.py, никаких архитектурных правок. Содержание 1:1 как в v1, длина каждого пузыря ≤ 4096 символов (Telegram limit), складные блоки расширяют ёмкость до ~10000. Объём работ — ~1 день одной задачей.
