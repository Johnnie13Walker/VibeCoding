Технический долг — что нужно для полноценного анализа чатов

Wazzup сообщения — REST API не отдаёт содержимое

В Belberry подключён коннектор WAZZUP (WhatsApp + Telegram + Instagram + Avito + Viber + Max) через imconnector. По данным Льва Петровича: вчера было 22 чата (Деговцова 6, Семенихин 5, Исаева 7, Вострецов 4). Но через стандартный REST-API Bitrix24 содержимое этих чатов недоступно:

  • imopenlines.session.list → метод не найден
  • imopenlines.dialog.get → ACCESS_ERROR
  • crm.activity.list с фильтром по PROVIDER_ID=IMOL_SESSION/OPENLINE → 0 активностей
  • im.recent.list → возвращает только чаты текущего токен-пользователя (Ларисы Помогатор, бот)

Чтобы анализировать чаты, нужен один из путей:

  1. Wazzup API напрямую — токен у администратора Wazzup-аккаунта Belberry. Wazzup отдаёт все сообщения и контакты через REST. Это самый чистый путь.
  2. Webhook от Wazzup — на каждое новое сообщение Wazzup присылает webhook. Можно поднять собственный приёмник на VPS Cloudbot и складывать сообщения в свою БД для анализа SLA.
  3. Битрикс24-приложение с расширенными правами — installable app со scope imopenlines и im, токен админа портала. Тогда imopenlines.session.list работает.
Что я мог бы анализировать при доступе к Wazzup
(1) Время ответа менеджера на сообщение клиента (SLA-аналитика). (2) Сообщения клиентов без ответа > 1 часа в рабочее время. (3) Тональность диалогов (через LLM): клиент раздражён / удовлетворён. (4) Повторяющиеся вопросы клиентов, на которые менеджер не отвечает с первого раза. (5) Пропавшие лиды — клиент написал, но в CRM ни звонка, ни задачи не появилось. Это самый ценный источник опасных триггеров, потому что в чатах большинство «холодных» сигналов вылезает раньше, чем в звонках.
