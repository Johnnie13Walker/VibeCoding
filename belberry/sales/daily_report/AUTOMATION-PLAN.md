> Источник формата и правил: [DAILY-SALES-REPORT-SPEC.md](file:///Users/pro2kuror/Documents/Cloudbot-Vault/09-Projects/belberry-deal-analysis/DAILY-SALES-REPORT-SPEC.md). Эталон вёрстки: `отчеты/Сводка_продаж_2026-05-29.html`.

## Открытые блокеры (нужны от пользователя / при деплое)

1. **🔒 Защита ссылки** (отчёт = чувствительные данные: клиенты, суммы, метрики людей). По умолчанию — **неугадываемый токен в пути** (`/sales-report/<rand32>/2026-05-29.html`), nginx без листинга каталога. Альтернативы: basic-auth, IP-allowlist офиса. → подтвердить.
2. **🌐 Хостинг.** Нужен nginx + статика по домену/поддомену на VPS. Уточнить: есть ли поднятый веб и домен, или поднимать. URL вида `https://<domain>/sr/<token>/<date>.html`.
3. **🤖 LLM-ключ и бюджет.** Глубокий разбор = вызов Anthropic. Переиспользовать ключ/бюджет стека Лев Петровича? → подтвердить.
4. **Telegram chat_id** менеджерского чата + чей бот шлёт (можно от «Лев Петрович», токен уже есть на VPS).
5. Связь с текущей текстовой сводкой Лев Петровича 09:30 — **дополнять** или со временем заменить (по умолчанию пока дополняем, не трогаем).

## Архитектура

```
cron 09:00 МСК (Пн-Пт)
  └─ daily_report_runner.py
       1. calc_target_date()      — прошлый рабочий день (учёт выходных/праздников РФ)
       2. collect.py              — Bitrix REST: deals/stagehistory/activities/
                                     items(1048/1056/1106)/voximplant + Wazzup timeline
       3. enrich.py               — фото менеджеров (base64), транскрипты встреч (urlMachine)
       4. analyze_llm.py          — Anthropic: разбор встреч по чек-листу, цитата дня,
                                     системные выводы, Тигр дня, тексты "30 секунд"/"кого пинать"
       5. render.py               — сборка HTML по шаблону (style.css + секции)
       6. publish.py              — запись в /var/www/sales-reports/<token>/<date>.html
       7. notify.py               — Telegram sendMessage со ссылкой в чат менеджеров
  └─ лог в reports/ + алерт в DM при падении
```

Базируется на готовых скриптах из ручного прогона (`/tmp/sales_2905/`): `bx.py` (Bitrix-клиент), `collect.py`, `vox2.py`, `photos2.py`, `readsheets.py`, `gen*.py` (рендер). Их нужно причесать в модуль `belberry/sales/daily_report/src/`.

## Что автоматизируется как есть vs требует LLM

- **Детерминированно (без LLM):** все цифры шапки, зависшие сделки по стадиям и порогам, активность менеджеров (телефония→воронка), таблицы встреч/брифов/КП, отказы.
- **Через LLM:** глубокий разбор встреч по транскрипту (чек-лист защита/брифинг, ✅/⚠️/❌), цитата дня, системные паттерны, тексты «Главное за 30 секунд» и «Кого пинать», подпись Тигра дня.

## Рабочие дни и края

- Пн отчитывается за Пт. Учитывать праздники РФ (таблица/`workalendar`).
- Официальная сводка «Опер» по выходным не считается → в Пн-отчёте ранг ТМ по телефонии + пометка (уже в техдолге эталона).
- Если встреч/данных нет — секции схлопывать, не выдумывать (правило spec).
