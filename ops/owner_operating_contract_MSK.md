# Контракт Контекста Владельца (MSK)

Обновлено: 2026-03-05

## Цель
Держать систему в стабильном режиме без ручного контроля, с предсказуемой доставкой обязательных отчетов и уведомлений.

## Приоритеты
1. Стабильность и доставка обязательных отчетов/уведомлений.
2. Быстрая диагностика и устранение инцидентов.
3. Безопасность и управляемость доступа.
4. Развитие функций без деградации прод-сценариев.

## SLA отчетов
- Ежедневный статус: до 09:30 Europe/Moscow.
- Формат: `ОК / есть проблемы`.
- При проблемах: что сломано, причина, что сделано, что осталось, когда следующий чек.

## Обязательные интеграции
- Telegram (бот и чат для доставки).
- OpenAI (рабочий API-ключ).
- Sentry (токен + org + project).
- Notion (токен при включенных сценариях Notion).
- GitHub CLI (авторизация для задач по PR/CI).

## Границы допущений
- Нельзя считать интеграцию рабочей только по наличию файла/переменной; нужна проверка вызова.
- Нельзя считать задачу закрытой без smoke/логов/результата доставки.
- Все временные метки, расписания и отчеты только в Europe/Moscow.
