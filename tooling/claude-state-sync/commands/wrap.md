Ты выполняешь end-of-session сведе́ние («wrap»). Цель: чтобы НИЧЕГО важного из этой сессии
не осталось незаписанным в три слоя — **память проекта** (`~/.claude/.../memory/`),
**Obsidian** (STATUS + Daily) и **git**. Это автоматизация ручного «что не сохранено».

Фокус: **$ARGUMENTS** (если пусто — определи проект(ы) сам по cwd, упомянутым путям и
изменениям сессии). Работай кратко, отчитываясь по шагам. Язык — русский, время — МСК
(относительные даты «вчера/в четверг» переводи в абсолютные).

## 1. Собрать факты (git, детерминированно)
- Определи проект(ы), над которыми реально шла работа.
- По всем относящимся worktree (`git worktree list`): новые коммиты за сессию (`git log`),
  изменённые файлы, текущая ветка vs canon, `git status --short` (uncommitted/untracked).
- Зафиксируй для отчёта: что закоммичено / запушено / смёржено / задеплоено; что НЕ
  закоммичено; ветки, разошедшиеся с canon; feature-ветки, не влитые в main; стейл-worktree.

## 2. Память (`~/.claude/projects/.../memory/`)
- Найди соответствующий `project_*.md` (или `feedback_*` / `reference_*`) и **обнови** его:
  что сделано, **принятые решения и их «почему»**, **уроки/грабли**, новые прод-факты
  (домены, env-переменные, миграции, пути, ключи-где-лежат). НЕ дублируй то, что выводится
  из кода/git; не пиши эфемерное (in-progress, текущую беседу).
- Если знание новое и самостоятельное — создай новый файл по структуре (frontmatter
  `name`/`description`/`metadata.type`) + строку-указатель в `MEMORY.md`. Связи — `[[wikilinks]]`.
- Перед записью проверь, нет ли уже файла на эту тему — обновляй его, не плоди дубли.
  Неверные/устаревшие факты — исправляй/удаляй.

## 3. Obsidian (vault `/Users/pro2kuror/Documents/Cloudbot-Vault`)
- В `09-Projects/<project>/STATUS.md`: подними `updated:` (МСК) и добавь дату-секцию с итогом
  сессии (решения, выкаты, PR, открытые вопросы). Frontmatter и wikilinks — по CONTRACT.
- Допиши в `Daily/<сегодня>.md` (создай при отсутствии, с frontmatter): кратко что сделано +
  ссылки/PR. Не лей воду.
- Новые файлы в `09-Projects/<name>/` без необходимости НЕ создавай; `00-Inbox/` не трогай.
- Внутренние ссылки — Obsidian wikilinks `[[имя]]`; не ставь wikilink из vault на Layer-1 память
  (она вне vault) — упоминай словами/путём кода.

## 4. Git-гигиена
- Коммить **только vault-изменения**, атомарно, лишь свои файлы (чужие незакоммиченные правки
  не подмешивать), осмысленное сообщение + трейлер `Co-Authored-By: Claude ...`.
- Выведи список проблем + предложений: незакоммиченный код, untracked-мусор (с проверкой —
  это живое или мёртвые остатки удалённых фич), разошедшиеся/несмёрженные ветки.
- **ОПАСНОЕ НЕ ДЕЛАЙ САМ без явного «сделай сам»:** merge/push в `main`, force-push, прод-деплой.
  Вынеси отдельным блоком «нужно твоё решение» + готовые команды на paste.
- Push vault в `main` часто блокит harness → если так, дай команду `cd ~/Documents/Cloudbot-Vault && git push`.

## 5. Отчёт
Короткой таблицей: что сохранено в (Память / Obsidian / git) и что осталось на пользователе
(merge / push / деплой). Без воды. Если по какому-то слою сохранять было нечего — так и скажи.

## 6. Снять маркер
В самом конце удали маркер «не сведено», чтобы SessionStart больше не напоминал:
