# Чек-лист приемки Архитектора

Использовать при приемке инженерной задачи, migration или исправления сломанного контура.

## Контур и source of truth

- [ ] Канонический runtime-репозиторий определен.
- [ ] Канонический Git remote определен.
- [ ] Понятно, что является source of truth для кода, env, cron и server state.
- [ ] Зафиксировано, совпадает ли локальный контур с GitHub и сервером.

## Маршрутизация и дублирование

- [ ] Проверены реальные entrypoints сценария.
- [ ] Не осталось скрытого дублирования workflow, cron, shell wrapper или bridge.
- [ ] Для каждого затронутого сценария определен один канонический путь исполнения.
- [ ] Legacy-контуры имеют явный статус: `live`, `migration`, `archive` или `dead`.

## Проверки после изменений

- [ ] Запущены тесты, которые существуют для затронутого контура.
- [ ] Выполнен dry-run или deterministic smoke-check.
- [ ] Выполнена ручная проверка критичного сценария.
- [ ] Проверены логи, отчеты или runtime-артефакты после изменения.

## Аналитическая часть

- [ ] Названа корневая причина, а не только симптом.
- [ ] Зафиксированы архитектурные риски и условия отказа.
- [ ] Зафиксирован остаточный техдолг.
- [ ] Сформулирована целевая схема, а не только временный workaround.

## Критерий готовности

- [ ] Есть понятный критерий готовности для инженера.
- [ ] Есть критерий приемки для Архитектора.
- [ ] Понятно, что еще остается вне текущего scope.
