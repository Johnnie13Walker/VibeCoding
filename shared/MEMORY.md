# Shared Memory

## Назначение

`shared/` — общий контур AI Workspace. Сюда попадает то, что должно быть доступно всем доменам (`personal`, `belberry` и будущие): переиспользуемые шаблоны, правила, агенты, общая инфраструктура и общие принципы работы.

Этот файл — точка памяти shared-домена. На него ссылается `agents/prompt-master/memory-rules.md`. Если задача не привязана ни к `personal`, ни к `belberry`, `prompt-master` обязан проверить shared-память перед генерацией orchestration flow.

## Что хранится в shared

- `shared/templates/` — шаблоны: agent, architecture, integration, system-prompt, workflow.
- `shared/agents/` — общие агенты, доступные всем доменам.
- `shared/prompts/` — переиспользуемые промпт-блоки.
- `shared/docs/` — общая документация и архитектурные решения уровня workspace.
- `shared/integrations/` — общие коннекторы и адаптеры к внешним сервисам.
- `shared/automation/` — общие автоматизации, не привязанные к конкретному домену.
- `shared/data/` — общие данные и конфигурации.
- `shared/sandbox/` — площадка для общих экспериментов перед промоушеном в домены.

## Правила

### Что класть в shared

- Решение применимо к двум и более доменам.
- Решение стабильно и проверено в одном из доменов.
- Решение не содержит доменно-специфичных секретов, ключей и бизнес-логики.

### Что НЕ класть в shared

- Production-секреты любого домена.
- Бизнес-логику конкретного клиента (это `belberry`).
- Эксперименты в активной фазе обучения (это `personal`).
- Незакрытые TODO без ответственного.

### Промоушен решения в shared

Решение переносится из `personal` или `belberry` в `shared`, только если:

1. Оно использовалось не менее чем в одном домене на реальной задаче.
2. Прошло проверку `Ревизора` и `Испытателя`.
3. Из него удалены доменно-специфичные ссылки и секреты.
4. К нему написан минимальный README или встроенная документация.
5. Указан владелец (агент или человек), отвечающий за поддержку.

### Деградация и удаление

- Решение, не использовавшееся в shared 6 месяцев и не имеющее владельца, считается кандидатом на архив (`archive/`).
- Удаление из shared делается только через `Мастера` с явным file ownership и согласованием с обоими доменами, если решение использовалось в production.

## Связь с prompt-master

`prompt-master` обязан проверить `shared/MEMORY.md` перед orchestration, если:

- задача общая и не привязана к `personal`/`belberry`;
- задача затрагивает шаблоны, общие агенты или переиспользуемые промпт-блоки;
- задача требует выбора между «дублировать в домене» и «использовать общий компонент».

Если решение уже есть в shared, `prompt-master` обязан использовать его, а не создавать новое в домене.

## Текущее состояние

- `shared/templates/` содержит 5 базовых шаблонов: agent, architecture, integration, system-prompt, workflow.
- `shared/agents/` пуст, ждёт первого общего агента.
- `shared/prompts/`, `shared/docs/`, `shared/integrations/`, `shared/automation/`, `shared/data/`, `shared/sandbox/`, `shared/assets/`, `shared/apps/`, `shared/experiments/`, `shared/logs/` — пусты, заполняются по мере появления переиспользуемых решений.

## Запреты

- Не дублировать содержимое `personal/MEMORY.md` или `belberry/MEMORY.md` сюда.
- Не помечать домен `shared` для задач, у которых явно есть домен-владелец.
- Не использовать shared как промежуточный буфер «положу пока сюда» — для незрелых решений есть `personal/sandbox/`.
