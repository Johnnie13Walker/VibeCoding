# Постоянные Правила Агента (MSK)

Обновлено: 2026-03-05

## Цель
Снизить повторяемые ошибки и обеспечить стабильный результат между сессиями.

## Guardrails
1. Любое изменение завершается проверкой: тесты + smoke + анализ логов.
2. Если есть workflow в orchestrator, запускать задачу через него.
3. Не закрывать задачу при статусе `есть проблемы`; сначала устранить причину.
4. Все расписания, дедлайны и отчеты только в Europe/Moscow.
5. В каждом итоговом отчете фиксировать артефакты и критерий проверки.
6. При недоставке обязательного Telegram-статуса считать это инцидентом.

## Повторяемые ошибки и защита
1. Ошибка: запуск без актуального контекста.
   Защита: `context_snapshot` перед критичными изменениями.
2. Ошибка: расхождение SLA в документах и cron.
   Защита: `instruction_conflicts`.
3. Ошибка: шумные прогоны без закрытия первопричины.
   Защита: `daily_ops` + инцидент + `next_week_prep`.
