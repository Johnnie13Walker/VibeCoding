# Блок `Воронка в работе`

## Назначение

Короткий операционный срез по активной воронке: сколько сделок реально в работе, где есть движение, на каком объёме уже потерян контроль и какие денежные риски уже сидят внутри текущего pipeline.

## Что должно попасть в расчёт

- только активные сделки в работе;
- `Активные сделки` — весь текущий объём активной воронки;
- `С движением за последнюю неделю` — сделки с подтверждённым движением за рабочий период;
- `Без движения за последнюю неделю` — сделки, которые стоят в текущем окне контроля;
- `Этап договора` — сделки на позднем договорном шаге, если такой срез подтверждён;
- `Под риском` — сделки с накопленными сигналами риска;
- под `Под риском` только:
  - `Без следующего шага`;
  - `Без коммуникации > 14 дн.`;
  - `С просроченными задачами`.

## Правила блока

- Блок канонический для ежедневной сводки: рядом не нужен отдельный блок `Деньги` с теми же агрегатами.
- Порядок строк фиксированный.
- Денежные строки допустимы только там, где они уже предусмотрены форматом.
- Дополнительные списки сделок внутрь блока не вставлять.
- Если данных по блоку недостаточно, выводить короткое ограничение вместо фиктивных чисел.

## Эталонный формат

```text
📈 Воронка в работе
• Активные сделки: {deals_in_work} / <b>{pipeline_amount}</b>
• С движением за последнюю неделю: {moving_deals_last_week}
• Без движения за последнюю неделю: {stagnant_deals_last_week}
• Этап договора: {contract_stage_deals} / <b>{contract_stage_amount}</b>
• Под риском: {risk_deals} / <b>{risk_amount}</b>
  - Без следующего шага: {without_next_step_deals} / <b>{without_next_step_amount}</b>
  - Без коммуникации > 14 дн.: {stale_deals} / <b>{stale_amount}</b>
  - С просроченными задачами: {overdue_deal_task_deals} / <b>{overdue_deal_task_amount}</b>
```

## Вариант при нехватке данных

```text
📈 Воронка в работе
• Активные сделки: ограничение — нет подтверждённого полного среза active pipeline.
• С движением за последнюю неделю: ограничение — нет надёжного признака движения.
• Без движения за последнюю неделю: ограничение — нет надёжного признака простоя.
• Этап договора: ограничение — договорный stage-срез не подтверждён.
• Под риском: ограничение — нет полного подтверждённого расчёта риск-сигналов.
  - Без следующего шага: ограничение — поле или proxy следующего шага не подтверждены.
  - Без коммуникации > 14 дн.: ограничение — источник коммуникаций не подтверждён.
  - С просроченными задачами: ограничение — задачи или привязка к сделкам не подтверждены.
```
