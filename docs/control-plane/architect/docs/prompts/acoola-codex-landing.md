# Codex prompt: лендинг Acoola.Team

Используй этот prompt в Codex, когда нужно создать или доработать продакшен-готовый одностраничный лендинг digital-агентства **Acoola Team**.

Перед стартом проверь все места с пометкой `TODO: подтвердить`: это неподтверждённые цены, даты, формулировки или распределение команды. Если есть доступ к интернету и задача требует публикационной точности, проверь факты по официальным источникам Acoola.Team и партнёрским страницам. Неподтверждённые данные не превращай в категоричные утверждения.

## Роль

Ты работаешь как senior fullstack-разработчик и продуктовый дизайнер в Codex. Твоя задача - не только написать код, но и довести лендинг до проверяемого состояния: локальный запуск, визуальная проверка, Lighthouse, адаптивность, доступность и чистый отчёт по результатам.

Работай на русском языке. Коммиты, README, комментарии и пользовательские сообщения - на русском. Время и даты указывай в МСК (`Europe/Moscow`).

## Задача

Создай одностраничный лендинг digital-агентства **Acoola Team**. Сайт должен:

1. За первые 5 секунд объяснять, кто агентство и почему его выбирают.
2. Презентовать 8 услуг с УТП и ориентировочными ценами.
3. Показать 6 кейсов с конкретными цифрами выручки, ROI и стоимости лида.
4. Снять возражения через команду, сертификаты, NPS, отзывы и процесс.
5. Вести посетителя к одному из двух CTA: `Оставить заявку` или `Получить персональное предложение`.

Ощущение сайта: зрелое B2B-агентство, уверенно, плотно, без инфоцыганщины, с акцентом на цифры, доказательства и прозрачный процесс.

## Дизайн-направление

Стиль: премиальный технологический B2B. Не креативное агентство с конфетти, а системный подрядчик, которому доверяют проекты с деньгами, метриками и ответственностью.

Палитра:

- фон: `#0A0A0B`;
- поверхность: `#111114`;
- основной акцент: `#00E5C7`;
- основной текст: `#FFFFFF`;
- вторичный текст: `#7A7A85`;
- границы: `rgba(255,255,255,0.08)`.

Важно для Codex-дизайна:

- не превращай страницу в однотонную бирюзово-чёрную тему; добавь вторичные нейтральные и графитовые состояния, чтобы интерфейс не выглядел плоско;
- не используй декоративные шары, bokeh, случайные SVG-абстракции и псевдо-3D;
- hero должен сразу показывать бренд и предложение, а не маркетинговую заставку;
- карточки не вкладывай в другие карточки;
- все элементы должны сохранять стабильные размеры на mobile и desktop;
- текст не должен вылезать из кнопок, бейджей и карточек.

Типографика:

- H1: 64-80px на desktop, Manrope, 700;
- H2: 40-48px, Manrope, 700;
- H3: 24-28px, Manrope, 600;
- body: 16-18px, Inter, line-height 1.6;
- крупные цифры: 56-72px, Manrope, 800, акцентный цвет.

Интеракции:

- скролл-анимации через `whileInView` с `opacity` и `y` только если они не скрывают LCP-контент и не просаживают Lighthouse; иначе используй CSS transitions/hover без задержки первого рендера;
- hover на карточках: подсветка границы и лёгкий `translateY(-2px)`;
- count-up для статистики при появлении в viewport;
- карусели только по клику или свайпу, без autoplay.

Адаптивность:

- mobile-first;
- проверить iPhone SE, iPhone 12, iPad, desktop 1440 и 1920;
- на мобильном hero не должен занимать больше одного экрана так, чтобы CTA уходили слишком далеко вниз.

## Маркетинговые правила

1. Не использовать клише: `команда профессионалов`, `индивидуальный подход`, `качество и сроки`.
2. Цифры должны быть раньше эпитетов.
3. Каждое УТП должно закрывать конкретное возражение клиента.
4. Используй анти-обещания: не обещаем X, зато показываем Y.
5. Доказательства должны стоять рядом с утверждениями.
6. Тон: уверенный, деловой, современный. Не панибратский и не канцелярский.

## Технические требования

- `title` и `meta description`;
- Open Graph и Twitter Card;
- семантические теги `header`, `main`, `section`, `article`, `footer`;
- `lang="ru"`;
- Schema.org `Organization` с контактами;
- `next/image` для изображений;
- lazy loading для вторичных медиа;
- CLS < 0.1;
- aria-label для интерактивных элементов;
- видимые focus-стили;
- навигация с клавиатуры;
- заглушка Яндекс.Метрики через `METRIKA_ID`.

## Что не делать

- Не использовать Material UI, Chakra и другие готовые UI-библиотеки.
- Не использовать стоковые иконки вроде handshake, лампочки и мишени.
- Не делать autoplay-слайдеры.
- Не вставлять фейковые цифры.
- Не использовать emoji в основном контенте.
- Не делать декоративную landing-заставку вместо рабочего первого экрана.

## Критерии готовности

1. Все 10 секций реализованы и наполнены.
2. Проект открывается локально без ошибок.
3. `npm run lint` и `npm run build` проходят.
