# Чеклист после изменений

Использовать после каждого изменения системы.

## Обязательно

1. Запустить тесты.
2. Проверить код возврата команд.
3. Пройти ручной smoke-test критичных сценариев.
4. Проверить логи сервиса после изменения.
5. Убедиться, что не появились новые ошибки.
6. Обновить `docs/STATUS.md`.

## Минимальный шаблон отчёта

- Что изменено
- Какие команды запускались
- Какие ручные сценарии проверены
- Какие логи просмотрены
- Итог: успешно / есть блокер
