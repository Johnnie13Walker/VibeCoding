# Ежедневный health-check Cloudbot

Все времена и отчёты вести в `Europe/Moscow`.

## Сервисы

- [ ] Основной backend запущен
- [ ] Telegram-бот запущен
- [ ] Планировщик задач работает
- [ ] Очереди задач работают
- [ ] Reverse proxy / web ingress работает

## Интеграции

- [ ] OpenAI доступен
- [ ] Bitrix24 доступен
- [ ] Todo-провайдер доступен
- [ ] WHOOP доступен
- [ ] Web Search доступен
- [ ] Все известные интеграции отражены в основном блоке `API и интеграции`
- [ ] Давние интеграции не попали повторно в блок `Новые API`

## Плановые задачи

- [ ] Cron-задачи выполнялись по расписанию
- [ ] Ночные задачи завершались без критичных ошибок
- [ ] Ежедневные отчёты были отправлены
- [ ] Триггерные уведомления приходили

## Логи

- [ ] Нет критичных ошибок за последние 24 часа
- [ ] Нет циклических падений сервисов
- [ ] Нет ошибок авторизации во внешних API
- [ ] Нет признаков лимитов или rate limit

## Пользовательские возможности

- [ ] Статус user-facing сценариев показан отдельно от статуса API
- [ ] `Web Search provider` не спутан с `Web search для Ларисы`
- [ ] Если поиск у Ларисы не настроен, это отражено как `not configured`/ограничение capability, а не замаскировано под `API OK`

## Итоговый статус

- [ ] Пользователю отправлен один итоговый утренний отчёт, а не несколько дублей
- [ ] Использован визуальный каркас `🟢 СЕРВЕР СТАБИЛЕН` / `🔴 ЕСТЬ ПРОБЛЕМЫ`
- [ ] В отчёте есть блоки: `Итог`, `Хост`, `OpenClaw / OpenCloud`, `API и интеграции`, `Пользовательские возможности`, `Планировщик и доставка`, `Безопасность`, `Наблюдения / техдолг`, `Индекс здоровья`, `Время отчёта`
- [ ] Если есть проблемы, перечислены проблемы, причина, исправление и следующий чек
- [ ] Деградированная диагностика (`missing scope: operator.read`) не спутана с недоступностью gateway
- [ ] Единичный self-healed cron error не спутан с активным инцидентом
- [ ] Если у job идёт более свежий run со статусом `running`, старый `error` не объявлен активной поломкой до завершения текущего прогона
- [ ] `multi_user_heuristic` классифицирован как риск/техдолг либо как подтверждённая проблема, но не безусловно
- [ ] Доступное обновление OpenClaw вынесено как maintenance backlog, если нет текущей деградации
