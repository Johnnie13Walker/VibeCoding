# Belberry PPC: короткий коммерческий аудит по поисковым запросам

Период: 22.03.2026-21.04.2026  
Источник: `2026-03-22_2026-04-21_searchquery_e-16726824.xlsx`  
Важно: столбец `Конверсии` в файле не расшифрован по `goalId/goalName`, поэтому ниже он используется только как техническая метрика. Реальные лиды отдельно не подтверждены.

## 1. Итог

По поисковым запросам потрачено 15 964,93 руб., получено 128 кликов и 94 технические конверсии. Но бизнес-лиды не подтверждены: ранее по бизнес-контуру был виден только 1 целевой визит. Поэтому вывод коммерческий: кампания покупает много кликов по запросам, которые не соответствуют дорогому B2B-офферу Belberry.

Главная причина просадки не в интерфейсе Директа, а в несоответствии трафика офферу. В поиске смешаны:

- дорогие общие запросы на разработку сайтов;
- DIY/бесплатные запросы;
- запросы про конструкторы Tilda/Wix/Nethouse/WordPress;
- образовательные/курсовые запросы;
- медицинские, но не обязательно коммерческие запросы.

Такой трафик может создавать дешевые технические события, но не обязан создавать лиды, встречи и сделки.

## 6. Правильное решение

Сейчас нельзя масштабировать кампанию. Нужно сначала сделать структуру, где каждый сегмент отвечает за свою экономику.

Обязательное разделение:

1. `Search / Medical Website / Hot / MSK-SPB` - только коммерческие запросы клиник и стоматологий.
2. `Search / General Website / Test` - общий спрос на разработку сайтов, с лимитом.
3. `Search / DIY-Free / Excluded or Low-cost test` - бесплатные/конструкторы, по умолчанию в минус.
4. `Search / Courses-Info / Excluded` - курсы, обучение, информационный спрос.
5. `Network / Retargeting` - отдельно от поиска.
6. `Network / Cold / Test` - отдельно, с отдельным бюджетом.

CRM должна получать минимум: запрос, кампания, группа, объявление, посадочная, goalId, goalName, телефон/email, статус лида, причина отказа, сделка, выручка. Без этого невозможно считать CPL, CPQL, CAC и ROMI.
